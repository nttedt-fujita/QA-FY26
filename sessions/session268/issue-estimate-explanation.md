## 見積もりの考え方

### 1. 「総Issue数」と「アクティブ数」の違い

**総Issue数**:
- プロジェクト全体で作成するIssueの総数
- 完了したものもアーカイブしたものも含む

**アクティブ数（最大）**:
- **同時に進行中**のIssueの最大数
- Open, In Progress, Blocked等のステータスのIssue
- **これがFreeプランの250制限の対象**

**例**:
```
Phase 1のSIPOC作成（総4 issue）:
- Issue 1: 全体フローSIPOC → 完了後アーカイブ
- Issue 2: 受入検査詳細SIPOC → 完了後アーカイブ
- Issue 3: 組立検査SIPOC → 未着手
- Issue 4: 出荷検査SIPOC → 未着手

アクティブ数（最大）: 2個（Issue 3, 4が同時に進行中）
```

### 受入検査改善（Phase 1-3）合計

| Phase | 総Issue数 | アクティブ数（最大） |

| Phase 1 | 28-33 | 14 |

| **合計** | **55-60** | **41** |

**41の内訳**:
- Phase 1の最大アクティブ: 14
- Phase 2の最大アクティブ: 14
- Phase 3の最大アクティブ: 13

**注意**: これは「Phase 1, 2, 3が全て同時に進行中」という最悪ケースの想定。実際にはPhaseは順番に実施するため、**同時にアクティブになるのは1つのPhaseのみ（14）**。

**ただし、Phaseをまたぐ場合**:
- Phase 1の一部（例: 隠れコスト特定 7個）が残っている
- Phase 2の一部（例: VSM作成 3個）を開始
- → 同時にアクティブ: 7 + 3 = 10

**安全マージン**: 最大41と見積もっているが、実際には14-20程度になる可能性が高い。

## 現実的な見積もり

**最悪ケース**: 71-101（全Phase・全ミッションが同時進行）

**現実的なケース**: 30-50（1つのPhase + 他ミッション）

**理由**:
- 受入検査改善はPhaseごとに順番に実施（同時に全Phaseは実施しない）
- 他ミッションも優先度を付けて順番に実施

**現実的なアクティブ数の内訳**:
- 受入検査改善（1つのPhase）: 14
- M1-GNSS開発: 10
- その他ミッション: 6-26
- **合計**: 30-50

**それでもBasicプランを推奨する理由**:
1. **安全マージン**: 30-50は余裕があるように見えるが、予期しないタスクが発生する可能性
2. **成長**: プロジェクトが拡大すると、アクティブissueも増加
3. **コスト**: 年$360-600は許容範囲内
