# 複数台検査フロー要件

作成: Session 205 (2026-03-16)

---

## 1. 要件サマリー

| 項目 | 決定 |
|------|------|
| 検査方式 | 一括・個別の両方対応 |
| DB登録タイミング | 検査完了時に自動登録（現行踏襲） |
| DB登録順 | USBポート順（ttyUSB0 → 1 → 2） |
| 検査後フロー | 結果表示 → 点滅で識別 → 取り外し |

---

## 2. 検査方式

### 一括検査

- 接続中の全デバイスを一気に検査
- 「全台検査開始」ボタン

### 個別検査

- 任意の1台を選んで検査
- デバイスごとに「検査開始」ボタン

### 現行との違い

- 現行: 1台のみ対応
- 新規: 複数台対応（一括/個別）

---

## 3. DB登録

### タイミング

検査完了時に**自動登録**（現行踏襲）

### 登録順

USBポート順で登録:
1. ttyUSB0
2. ttyUSB1
3. ttyUSB2

### 理由

- シンプル
- 順番を気にする必要がない
- 現行の自動登録フローを変更しない

---

## 4. 検査後フロー

```
1. 複数台接続（例：3台）
2. 画面に一覧表示（シリアル番号付き）
3. 「全台検査開始」または「この1台を検査」
4. 検査実行
5. 検査完了 → 自動でDB登録（USBポート順）
6. 結果表示（OK/NG）
7. 点滅ボタンで識別 → シール貼り/対処 → 取り外し
8. 次のロットへ
```

## 5. 実装方針

### Phase 1: 一括検査対応

1. 複数デバイスの並列検査
2. 結果の集約表示
3. USBポート順でDB登録

### Phase 2: 個別検査対応

1. デバイスカードに「検査開始」ボタン追加
2. 検査中は他デバイスの操作を制限？（要検討）
