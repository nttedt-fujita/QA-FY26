## 1. PDFについて（何の文書か）

**ファイル名**: u-blox F9 HPG 1.32 Interface Description (UBX-22008968)
**性格**: **UBXプロトコル仕様書（チップとソフトの通信API仕様）**

u-centerが内部でF9Pチップと通信するときに使うメッセージ形式を定義した文書。
「u-centerの操作マニュアル」ではなく「u-centerの裏側の仕様書」。

→ **Phase 1では直接使わない。Phase 2（ツール設計）で重要になる。**

### Phase 2以降に使う主要メッセージ（索引）

| メッセージ | ページ | 取得できるデータ | 評価用途 |
|-----------|--------|----------------|---------|
| UBX-NAV-PVT | p.145 | fixType, **carrSoln**, numSV, hAcc, vAcc, pDOP | 最重要。RTK状態・精度・衛星数を一括取得 |
| UBX-NAV-HPPOSLLH | p.140 | hAcc/vAcc（0.1mm精度） | mm単位の高精度測位 |
| UBX-NAV-STATUS | p.155 | gpsFix, carrSoln, **ttff**（初期FIX時間ms） | 機体準備時間の自動計測 |
| UBX-NAV-SAT | p.150 | cno (dBHz), elev, qualityInd（衛星ごと） | 受信感度の自動記録 |
| UBX-NAV-SIG | p.152 | cno (dBHz)（L1/L2信号別） | L1/L2ごとの受信感度 |
| UBX-MON-RF | p.132 | **jammingState**, noisePerMS, agcCnt | ジャミング検出・ノイズ監視 |
| UBX-MON-SPAN | p.134 | 256点スペクトラムデータ | スペアナの自動取得 |

### RTK FIX状態の定義（重要）

`UBX-NAV-PVT.flags` の bits 7-6（`carrSoln`）：

| 値 | 意味 |
|----|------|
| 0 | RTKなし（通常測位） |
| 1 | RTK Float（不安定） |
| **2** | **RTK Fixed（cm級精度 = 合格）** |

## 3. 全シートを通じた重要発見

### 発見①：No.5の異常が複数シートで一致

| シート | No.5の異常 |
|-------|-----------|
| 受信感度（3回目） | Q2 L1 = **13dBHz**（他は30〜48） |
| 受信感度（1回目） | Q7 L1 = 25（Refの41より-16） |
| 20260218 | RTK FIX **不可**（5分以上）|
| 20260218 | 飛行中衛星数 **19**（他は25〜27） |

→ **No.5は受信感度が低く、それがRTK FIX不可に直結している可能性が高い。**

### 発見②：内部設定の仕様合意が未完了

「小峰無線様と仕様の取り交わしが必要」→ 受入検査の「パラメータ確認」項目は、**先に小峰無線との合意事項を明確にしないと検査基準が作れない。**

### 発見③：QZSSのL2は全機体で受信不可

全衛星・全機体でQZSS L2 = 「ー」。F9Pの設定かアンテナ特性の問題か要確認。受入検査で「L2確認」する場合はGPS衛星を使う必要がある。

### 発見④：電池確認のHot start確認が未完了

No.1, No.3, No.5の屋外Hot start確認未記録。受入検査に「Hot start確認」を含めるには手順の確立が先。

### 発見⑤：スペアナは「合格基準」ではなく「比較」用途

シルクなし機体vs小峰GPS機体の比較に使用。製造ばらつき検出のためには**基準となるスペクトラムパターン**が必要（現状なし）。
