## 4. 今わかっている範囲で言えること

### 現状の痛み（推定）

```
掛川さん: 毎週日曜にPSI集計（工数?時間）
         → kintoneから手動で数字を転記
         → コメント欄に顧客別内訳を手書き（282件/シート）

現場全体: 実在庫が月末まで見えない
         → 理論在庫と実在庫のギャップが不明
         → 欠品 or 過剰在庫のリスク（実害エピソード未確認）
```

### 自動化で解消できそうなこと（技術的に）

| 施策 | 難易度 | 効果 |
|------|--------|------|
| kintone→PSI自動取得 | 中（API次第） | 掛川さんの週次集計工数削減 |
| 受入検査→GEN入庫反映 | 中〜高（GEN API次第） | 入庫記録の即時化 |
| QRコードによるロット管理 | 低（宇枝さん資料の通り） | トレーサビリティ確保 |

### まだ言えないこと

- 具体的な工数削減の数字（Q1が未回答）
- 導入の優先順位の根拠（Q2, Q3が未回答）
- 技術的な実現可能性（Q4, Q5が未回答）
