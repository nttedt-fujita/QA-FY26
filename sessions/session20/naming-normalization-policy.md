# 名寄せルール方針

**作成日**: 2026-03-06 (Session 20)
**ステータス**: 方針整理完了、実装は次回以降

---

## 背景

受入検査データには以下の表記揺れが存在する:
- **品名**: `Arm grommet` vs `Arm　grommet`（全角スペース）
- **検査内容**: `外観　寸法` / `外観寸法` / `外観・寸法` / `外観、員数`
- **作業者名**: 表記揺れの可能性あり

表記揺れを名寄せする際、**変換履歴を明確に残す**ことが重要。

---

## 方針

### 1. 名寄せルールファイル

名寄せルールは `mapping/` ディレクトリにCSVで管理する。

```
sessions/session20/
├── mapping/
│   ├── 品名_名寄せルール.csv
│   ├── 検査内容_名寄せルール.csv
│   └── 作業者_名寄せルール.csv
```

### 2. CSVフォーマット

| 列 | 説明 |
|---|------|
| `変換前` | 元のデータに存在する値 |
| `変換後` | 名寄せ後の正規化された値 |
| `適用理由` | なぜこの変換を行うか |
| `登録日` | ルール追加日 |

**例: 検査内容_名寄せルール.csv**
```csv
変換前,変換後,適用理由,登録日
外観　寸法,外観+寸法,全角スペースを+に統一,2026-03-06
外観寸法,外観+寸法,区切りなしを+に統一,2026-03-06
外観・寸法,外観+寸法,中黒を+に統一,2026-03-06
外観、員数,外観+員数,読点を+に統一,2026-03-06
外観，員数，寸法,外観+員数+寸法,複合を+に統一,2026-03-06
```

### 3. 適用方法

1. 分析スクリプトはまず**名寄せなし**で実行（現状）
2. 名寄せルールCSVが存在する場合は、読み込み時に変換を適用
3. 出力CSVには名寄せ後の値を使用
4. **元の値は生データCSVに残る**（追跡可能性を確保）

### 4. 実装時の注意点

- 名寄せルールは**追記のみ**（既存ルールの削除は禁止）
- ルール変更時は新しい行で上書き（登録日で最新を判断）
- 名寄せ前後の件数を比較して、意図しない統合がないか確認

---

## 名寄せ対象の優先度

| 対象 | 優先度 | 理由 |
|------|--------|------|
| 検査内容 | 高 | Method分析に直接影響。表記揺れが多い |
| 品名 | 中 | パレート分析に影響。部品種類数が減る |
| 作業者 | 低 | 表記揺れが少ない可能性 |
