# Session 152 サマリー

**目的**: 統合APIのタイムアウト・パースエラー問題を解決

---

## 実施内容

### 1. 問題の再現確認

統合API (`/api/gnss-state`) を実行すると:
- NAV-STATUS: タイムアウト
- NAV-SIG: タイムアウト
- MON-SPAN: パースエラー（前のメッセージの応答を受信）
- MON-RF: パースエラー（前のメッセージの応答を受信）

単発APIは全て成功 → 統合API固有の問題と判明。

### 2. 原因特定

**テストスクリプトで各APIの応答時間を計測**:
- NAV-SAT: 0.2秒
- NAV-STATUS: **1.5秒**
- NAV-SIG: 0.4秒
- MON-SPAN: **1.0秒**

**根本原因**: 統合APIのタイムアウト500msが短すぎる。NAV-STATUS/MON-SPANは1秒以上かかる。

### 3. 修正

`gnss_state_api.rs` の `poll_and_parse!` マクロでタイムアウトを **500ms → 2000ms** に延長。

### 4. 検証

**100回連続テスト: 成功100 / 失敗0**
