## 3. スライド構成（案）

### スライド1: タイトル

- AI検査システム 1次報告
- 日付、報告者（藤田）

### スライド2: 背景・きっかけ

- 宇枝部長からの問い合わせ
- 1万台製造の将来構想
- 出典: Session 242

### スライド3: 調査結果サマリー

- AI検査の位置づけ: M3/M4の一部として統合
- 推奨サービス: Amazon Bedrock（学習データ不要）
- 対象: プロポ・ペラの外観検査

### スライド4: コスト試算結果

- 損益分岐点（表）
  - 現状500台: 13-34年
  - 1万台: 1年以内
- 初期投資: 160-430万円
- 出典: Session 249

### スライド5: 懸念点

- 規模感（現状では効果出にくい）
- 判定基準の属人化（「個人主観」「都度判断」）
- 効果予測困難（学習データ量と精度の関係が不確実）
- 出典: Session 249-250

### スライド6: 今年度の提案

**メッセージ**: 小さく始める

| 項目 | 内容 |
|------|------|
| 今年度 | 原理検証プロトタイプ（1週間程度） |
| 目的 | 効果の確認、技術習得 |
| 本格導入 | 1万台規模が見えてから判断 |

### スライド7: 他ミッションとの兼ね合い

- 担当ミッション一覧（M1-A/B, M2, M3, M4）
- M4が最優先（既に流出問題発生）
- AI検査に時間を割くと他が遅れる
- 出典: Session 52（小笠原さんフィードバック）

### スライド8: 次のアクション

- [ ] 原理検証プロトタイプの時期調整（M4との兼ね合い）
