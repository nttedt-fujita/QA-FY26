# トークン効率化の改善記録

**日付**: 2026-03-19
**セッション**: Session 272

---

## 問題

**セッション開始時にトークンを28.7%消費**（57,327 / 200,000 tokens）

### 主な要因

| 要因 | トークン数 | 割合 |
|------|-----------|------|
| session-management スキル | ~10,000 | 17.4% |
| session-history (485行) | ~8,000 | 14.0% |
| linear-management スキル | ~4,000 | 7.0% |
| グローバル設定 | ~11,000 | 19.2% |
| その他 | ~24,327 | 42.4% |

---

## 根本原因

### 1. session-managementスキルの指示が曖昧

**問題の指示**:
- 「過去セッションの流れ・思考の変遷を把握する」
- 「プロジェクトの背景・経緯を理解する」

→ **何をすればいいか不明確**、全部読むと解釈される

### 2. session-historyのフォーマットが未定義

**問題**:
- 「2-3行の概要」と書いてあるが守られない
- 実際は各セッション20-50行になる
- 詳細な説明・ファイル一覧表を書いてしまう

**理由**: 歯止めがない

---

## 対策

### 1. session-managementスキルの修正

#### セクション2「セッション履歴ファイルを読む」の修正

**修正前**（曖昧）:
```markdown
- [ ] 過去セッションの流れ・思考の変遷を把握する
- [ ] プロジェクトの背景・経緯を理解する
```

**修正後**（具体的）:
```markdown
**目的**: 関連する過去セッションを見つけるため（インデックスとして使用）

- [ ] 各セッション1-3行の概要を確認（詳細は読まない）
- [ ] 関連するセッションがあれば、そのsession-summary.mdを読む

**重要**:
- session-historyは**インデックス**として使用
- 全セッションの詳細を読む必要は**ない**
```

#### セクション5「セッション履歴ファイルへの追記」の修正

**追加内容**:
1. **重要な情報の定義と配置先**（表で明示）
2. **厳格なフォーマット**（テンプレート提供）
3. **行数制限**（基本2-3行、最大10行）
4. **禁止事項**（詳細な説明・ファイル一覧表を禁止）

**修正ファイル**: `~/.claude/skills/session-management/SKILL.md`

---

### 2. session-historyの書き直し

**新フォーマット**:
```markdown
## Session XXX (YYYY-MM-DD)

**概要**: [1行、何をやったか]

**重要な成果**（あれば）:
- ADR-XXX: [タイトル] → `docs/adr/.../ADR-XXX-....md`
- [技術資料名] → `docs/missions/.../xxx.md`

**詳細**: [session-summary.mdへのリンク]
```

**実施結果**:
- `sessions/session-history/session-261-270.md` を書き直し
- **485行 → 132行（73%削減）**

---

## 削減効果（推定）

| 項目 | 改善前 | 改善後 | 削減量 |
|------|--------|--------|--------|
| session-management スキル | ~10,000 tokens | ~5,000 tokens | **-5,000 tokens** |
| session-history | ~8,000 tokens | ~2,000 tokens | **-6,000 tokens** |
| **合計削減** | - | - | **-11,000 tokens（-19.2%）** |

**改善後の推定**: 57,327 tokens → **46,327 tokens（23.2%使用）**

---

## 重要な原則（確立）

### 1. session-historyはインデックス

- 詳細を書かない
- 配置先パスを記載
- 関連セッションを見つけるために使う

### 2. 重要な情報は適切な場所に配置

| 情報の種類 | 配置先 |
|-----------|--------|
| 設計判断 | `docs/adr/` |
| 重要な発見・教訓 | プロジェクトCLAUDE.md |
| 技術調査結果 | `docs/missions/*/` |
| 新スキル・ルール | `~/.claude/skills/` or `rules/` |

### 3. 長くなる重要な情報は別ファイルに

- session-historyには書かない
- 適切な場所に配置してリンク
