## 背景

社長との意見交換（2026-03-25夕方）を前に、GEN・kintoneの制約が確定し、
当初のPSI集計自動化計画が破綻していることを整理した。

---

## 確定した制約

### GEN

| 項目 | 内容 |
|------|------|
| API状態 | β版（サポート対象外） |
| 取得可能データ | 品目マスタ、取引先マスタ（CSVのみ） |
| エンドポイント | `GET /api/itemMaster/csv`、`GET /api/customerMaster/csv` |
| 取得不可 | 受注・出荷・在庫等のトランザクションデータ |
| 各画面のDBアクセス | 不可 |
| 今後のAPI拡張 | 未定（積極的ではない） |

**出典**: GENベンダーからのメール（2026-03-25受領）

### kintone

| 項目 | 内容 |
|------|------|
| 契約形態 | NTT東日本の基盤上で利用中 |
| API利用 | NTT東日本の許可が必要 |
| 添付ファイルAPI | 取得不可 |
| カスタマイズ | NTT東製ノーコードツールのみ |
| 引っ越し検討 | 和田さん・宮崎さんが対応中（具体的計画は未定） |

**出典**: Session 295/298の整理、宮崎さん打ち合わせ（2026-03-24）

---

## 計画への影響

PSI集計自動化（QA-12）の前提が**両方とも崩壊**:
- GEN → トランザクションデータ取得不可
- kintone → API利用に制約あり

## PSI集計に絞った検討

### 現状の作業フロー

掛川さんが毎週日曜2h（年間104h）で手動集計:
- kintoneからレコードNo・発送先・発送日を手動転記
- GENから数値を画面で目視して拾う
- コメント欄に顧客別内訳を手書き（282件/シート）

### PSIに必要なデータ

| データ | 種類 | GEN APIで取れるか |
|--------|------|-------------------|
| 出荷数・受注数 | トランザクション | ❌ |
| 在庫数 | トランザクション | ❌ |
| 品番・品名 | マスタ | ✅ |
| 取引先情報 | マスタ | ✅ |

→ **マスタだけでは集計に必要な数値は取れない**

## 全体の方向性（検討中）

| # | 方向性 | メリット | リスク |
