## 3. M3/M4要件への適合性評価

### ルックアップ機能（lot_id紐付け）

| 要件 | kintoneでの実現 | 評価 |
|------|----------------|------|
| M3の入荷ロット → M4の不良レコードでlot_id参照 | ルックアップフィールドで実現可能 | ○ |
| 関連レコード一覧表示 | 関連レコードフィールドで実現可能 | ○ |
| 双方向のリアルタイム参照 | 関連レコードはリアルタイム表示 | ○ |

**制約**:
- ルックアップは**取得時点のスナップショット**。参照元が更新されても参照先は自動更新されない
- API経由でルックアップフィールドを更新する場合、**重複がない状態**でないと動作しない

### 集計・分析機能

| 分析手法 | kintone標準 | プラグイン/外部 | 評価 |
|---------|------------|---------------|------|
| 棒グラフ・折れ線・円グラフ | ○ | - | ○ |
| クロス集計 | ○ | - | ○ |
| **パレート分析** | **△ 直接は不可** | プラグインまたはPython連携 | △ |
| **SPC管理図** | **× 不可** | Python/外部ツール必須 | × |
| **Cpk計算** | **× 不可** | Python/外部ツール必須 | × |
| 時系列トレンド | △（折れ線グラフで簡易的に） | - | △ |

**結論**: 基本的なグラフは作れるが、**品質管理に必要な統計的分析（SPC、Cpk）はkintone単体では不可能**

### API連携

| 項目 | 内容 |
|------|------|
| REST API | あり（スタンダードコースのみ） |
| Python連携 | 可能（pykintoneライブラリ等） |
| レートリミット | 同時接続100件/ドメイン、1日10,000リクエスト/アプリ |
| Webhook | あり |

**評価**: API経由でPythonからデータ取得し、高度な分析を外部で行うことは**可能**。
5人・数十件/日の利用ではレートリミットに達する心配はない。

### カスタマイズの限界

| 項目 | 可否 |
|------|------|
| フォームレイアウト | ○（ドラッグ&ドロップ） |
| JavaScript/CSS | ○（スタンダードのみ） |
| プラグイン追加 | ○（スタンダードのみ、有料プラグインあり） |
| **複雑な計算式** | **△（基本的な四則演算のみ）** |
| **アプリ間をまたぐ集計** | **× 標準機能では不可** |
| **独自のデータベース設計** | **× リレーショナルDB的な設計は限定的** |

## 4. 製造業での活用事例

Web調査で確認された事例:

| 企業/事例 | 用途 | ポイント |
|----------|------|---------|
| 芙陽工業 | 製造不具合管理 | 現場からスマホで報告、写真添付、自動集計でトラブルゼロ達成 |
| 富士フイルムBI連携 | 製造業向け見える化 | kintone + 外部連携で品質管理を実現 |
| 各種導入事例 | 受注・在庫・品質管理 | ノーコードで現場主導のアプリ構築 |

**共通パターン**: 記録・報告・共有はkintoneで十分。分析の深掘りは外部ツール併用が多い。

## 5. 総合評価

| 評価軸 | スコア | コメント |
|--------|--------|---------|
| lot_id紐付け | ○ | ルックアップ + 関連レコードで対応可能 |
| データ入力・閲覧 | ◎ | タブレット対応、ノーコードでフォーム作成 |
| 基本集計・グラフ | ○ | 棒・折れ線・円グラフ、クロス集計OK |
| パレート分析 | △ | 標準では不可、プラグインまたは外部連携 |
| SPC・管理図 | × | kintone単体では不可能 |
| API連携 | ○ | Python連携可能（スタンダードコースのみ） |
| 保守性 | ◎ | サーバ管理不要、ノーコードで変更可能 |
| コスト | △ | 10ユーザー最低契約、既存契約なら追加コスト小 |
| 柔軟性 | △ | RDB的な設計は限定的、複雑な要件には不向き |
