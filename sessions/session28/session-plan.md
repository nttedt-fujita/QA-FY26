# Session 28 計画

**目的**: 品質管理視点を自動反映する仕組みの整備 + As-Is図の改修

---

## 背景

Session 27で発覚した問題:
- Session 25で品質管理フレームワーク（IQC/PQC/OQC、ロット、AQL、8D等）を調査済み
- しかし、Session 27のAs-Is図作成時にその視点が反映されなかった
- **調査結果が「自動的に作業に反映される仕組み」がない**ことが根本原因

---

## やること

### 1. 新スキル「qa-design-review」の作成（優先度: 高）

**目的**: M3/M4関連の設計・レビュー時に品質管理の視点を自動チェック

**スキルの内容案**:
- IQC/PQC/OQCの位置づけ確認
- ロット/トレーサビリティの考慮
- AQL/抜取検査の基準
- 8Dレポート（問題→原因→対策→効果確認）の観点
- M3/M4の紐づき（共通の部品マスタ、ロットID）

**参照資料**:
- [Session 25: 品質フレームワーク調査](../session25/quality-framework-research.md)

**配置場所**: このプロジェクト固有のスキルとして、まずはプロジェクト内に作成
- `skills/qa-design-review/SKILL.md` または
- `.claude/skills/qa-design-review/SKILL.md`（個人グローバル）

### 2. CLAUDE.mdに誘導記述を追加

**追加内容案**:
```markdown
## 品質管理設計ルール

M3/M4関連の設計・図作成時は `qa-design-review` スキルを参照すること。
```

### 3. As-Is図の改修

Session 27で作成した [as-is-model.drawio](../session27/as-is-model.drawio) に以下を追加:
- IQCとしての位置づけ（工程全体の中での位置）
- M3/M4の紐づき（共通の「部品」で連携）
- 現行Excelで「できていること/できていないこと」の対比
- ロット/トレーサビリティの欠落を品質管理観点で説明

または、別途「品質管理観点でのギャップ分析図」を作成
