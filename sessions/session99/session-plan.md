# Session 99 計画

**目的**: UBX通信タイミング問題のデバッグ・根本原因特定

---

## 背景

Session 98で検査画面を実装し、バッファドレイン+50ms待機を追加したが、
タイミング問題は完全には解決していない。

**現象**:
- 5項目中、ポート設定は高確率で成功
- 通信疎通、FWバージョン、シリアル番号はエラーになることが多い
- ボタンを押すたびに結果が変わる

---

## やること

### 1. デバッグ手法の検討

- ログ出力の追加（send_ubx, receive_ubx, drain_buffer）
- 送受信データのダンプ（バイト列をhex表示）
- タイミング計測（各操作の所要時間）

### 2. 根本原因の特定

調査観点:
- [ ] 応答が届いていないのか、届いているがパース失敗なのか
- [ ] 項目ごとの成功率の差は何に起因するか
- [ ] drain_bufferで正規の応答を消費していないか
- [ ] タイムアウト設定は適切か

### 3. 対策の実装

原因に応じて:
- 待機時間の調整
- リトライ機構の追加
- UBXフレームの完全性チェック

---

## 追加要望（後回し可）

- トランザクション/ロールバック機能
- テストデータ一括削除機能
