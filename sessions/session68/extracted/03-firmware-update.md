```
ファームウェアの更新
本機のファームウェアを更新する手順を説明します。
ファームウェアは、AS-DT1_uud_V1.00.f.bin といったファイル名で提供されます。
USB CDC 経由でも、8 ピンコネクターUART 経由でもアップデートを行うことができます。
USB CDC 経由の場合は、UPDATE モードで、UART 経由の場合は、UARTUPDATE モードでアップデートを行います。
UARTUPDATE モードでファームウェアを更新した場合でも、再起動後の測距モードはUPDATE モードになります。また、UART 接
続時のビットレート設定も初期化されます。再起動後もUART をご利用になる場合は、ファームウェア更新後、本機を再起動する前
に以下を設定してください。
• fluart コマンドでビットレートを設定
• flstart コマンドでUARTUPDATE またはEXTUART を設定
アップデートパッケージ
アップデートパッケージのバージョンを確認する方法について説明します。
アップデートパッケージ（AS-DT1_uud_*.f.bin ファイル）は下記のような構成となっています。
各領域のバージョン情報は、表のアドレスからASCIIZ 文字列で入っています。
サンプルプログラムによる更新
AS-DT1_sample_application.exe を使用してファームウェアの更新を行うことができます。更新方法についてはユーザーズガイドを
ご覧ください。
Python API による更新
サンプルプログラムソースコードas_dt1_api_update.py のas_dt1_api_update クラスにポート番号、ビットレート、ファイル名を
引数で渡して実行することで、ファームウェアの更新を行うことができます。
update 関数は呼び出されると完了するまで戻ってきません。アップデートが完了または強制終了されると、callback で指定された関
数が呼び出されます。進捗状態は別スレッドでget_progress 関数を呼び出すことで取得することができます。別スレッドから
set_stop 関数を呼び出すことで書き込みを強制終了させることができます。
本体コマンドによる更新
ファームウェアの更新の前にflstart コマンドで、測距モードをUPDATE またはUARTUPDATE モードに切り替えてください。
ファームウェアの更新が必要なアドレス領域は、下表に示したバンク2 ～バンク6 の領域となります。
```

## Page 38

```
38
バンク番号0、1 のboot およびupdate の書き換えは不要です。製造でのみ書き込みを行える領域でユーザーがupdate モードで書
き換えることはできません。
書き込まれたファームウェア詳細バージョンの確認
本機に書き込まれたファームウェアの詳細な情報を確認することができます。
findprg コマンドにより、バンクごとに書き込まれているファームウェアのバージョンを表示することができます。
findprg コマンドを実行すると、下記のような表示となります。
バンクが正しく書き込まれていない場合、バージョンは表示されません。
書き込み手順
プログラムすべて書き込んだ後に各バンクの先頭にヘッダー情報を書き込むことで、ファームウェアの更新時に、不完全なプログラ
ムが起動しないようになります。
書き込みは下記の手順で行います。
1
ファームウェアファイルAS-DT1_uud_V1.00.f.bin をバイナリーモードで開く。
2
アップデートパッケージ（37ページ）に記載されているヘッダー情報が、期待するファームウェアであることを確認する。
書き込み開始ファイル位置を0、書き込み先バンクを2、書き込み先アドレスを0x0とします。
3
eraseコマンドを使用して、書き込み先バンクを消去する。
4
wrtコマンドを使用して、ファームウェアファイルのファイル位置+0x100～0x7ffffを書き込み先アドレス0x100～0x7ffffに256
バイト単位で書き込みを行う。バンク2に限り0x7efffまでの比較を行う。
5
wrtコマンドを使用して、ファームウェアファイルのファイル位置+0x0～0xffを書き込み先アドレス0～0xffに書き込みを行う。
6
ファームウェアファイルのファイル位置を+0x80000、書き込み先バンクを+1とする。
7
バンク7を書き込むまで4～7を繰り返す。
書き込み確認手順
書き込みが正常に行われたか比較を行うには、下記の手順を行います。
1
比較開始ファイル位置を0、比較先バンクを2、比較先アドレスを0x100000 とする。
2
vrfコマンドを使用して、比較開始ファイル位置+0～0x7ffffと比較先アドレス0～0x7ffffを256バイト単位で比較を行う。バンク2
に限り0x7efffまでの比較を行う。
3
OKが帰ってくることを確認する（NGが帰ってきた場合、再度書き込みを行う）。
4
比較開始ファイル位置を+0x80000、比較先バンクを+1とする。
5
バンク7を比較し終わるまで2～4を繰り返す。
eraseコマンド
引数：引数で指定されたプログラム領域を消去する [2 ～6]
ご注意
> findprg
prog 0 exist dToF boot Version 1.000
prog 1 exist dToF upd Version 1.000 B128
spad 2 exist Version 1.000 B128 REG 1.0.0.z.6
prog 3 exist dToF cdc Version 1.000 B128
prog 4 exist dToF uart Version 1.000 B128
prog 5 exist dToF hist Version 1.000 B128
prog 6 exist dToF int Version 1.000 B128
erase 2
```
