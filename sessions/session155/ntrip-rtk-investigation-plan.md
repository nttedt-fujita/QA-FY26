# NTRIP/RTK分離テスト計画

作成: Session 155 (2026-03-13)

## 背景

現在の屋外検査フローでは、NTRIP接続と検査開始が独立した操作になっている。

```
現状:
1. 設定画面でNTRIP接続ボタンを押す
2. 屋外検査画面に戻る
3. 検査スタート

問題:
- NTRIP接続中は内部でずっとデータのやり取りが行われている
- 他の処理と同時並行できるか不明
- RTKのときだけ必要な分だけ処理するべきでは？
```

## 核心の疑問

> **NTRIP接続中（RTCMデータ転送中）に、定期出力（NAV-SIG, NAV-STATUS等）の読み取りは並行して動作できるか？**

## TDDスキル改善の方針

（詳細は `tdd-workflow-tradeoff-memo_20260313.md` 参照）

### 問題

各フェーズで「抜け漏れはないか？」と聞くと、毎回何か見つけてくる。しかしその中には「本当に重要な見落とし」と「聞かれたから絞り出しただけのもの」が混在している。

### 改善の方向性

「抜け漏れありますか？」ではなく「**この判断基準に照らして、優先度の高い見落としはありますか？**」に変える。

### フェーズ別の判断基準（案）

| フェーズ | スコープの限定 |
|---|---|
| 文脈理解 | 「このフィーチャーが解決するユーザーの問題は何か」だけに絞る |
| 振る舞い記述 | 「公開インターフェースから観測できる振る舞いだけ」に限定する |
| テストリスト | 「正常系・異常系・境界値の3カテゴリで各2〜3件」のように上限を設ける |
