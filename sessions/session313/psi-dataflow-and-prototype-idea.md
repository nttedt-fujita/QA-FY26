## 2. 藤田さんのアイデア（プロトタイプ構想）

### コンセプト

**関連情報を集約する小さなDBシステム**を作り、CSVインポートで入力・更新を自動化する。

### イメージ

```
【初回のみ】
[群馬通商 Excel] ──→ インポート ──→ [小さなDB] に初期データとして取り込み
  ※消し込みのみで新規追加なし → 以降のインポート不要

【定常運用】
[kintone CSV] ──→ インポート ──→ [小さなDB] ──→ PSI集計結果を自動出力
  ※CSVをインポートしたら、入力・更新が勝手にされる
```

### 群馬通商Excelの扱い

- 消し込み（出荷済み等で減る）のみ。新しい情報の追加はない
- → 初回にDBへ取り込めば、以降は不要
- → **kintone CSVが実質唯一のインプット**になる

## 4. 判断基準フレームワークでの位置づけ

### ペースレイヤード戦略での分類

- PSI集計 = **Systems of Innovation**（分析・集計、要件が変わりやすい）
- → **独立した小さなシステム**で作るのが妥当

### EAIパターン

- **DWH的アプローチ**に該当
- CSVインポート → DB集約 → 集計出力
- 業務システム（kintone, GEN）に影響を与えない

### 5C評価

| 観点 | 評価 | 根拠 |
|------|------|------|
| Consistency | 低（結果整合性で十分） | 週次集計なので即時整合不要 |
| Currency | 低（週次で十分） | 掛川さんの作業は週1回 |
| Cost | 低（小さなDB + CSV取込） | プロトタイプなので最小投資 |
| Competency | 低（集計は汎用機能） | 専門的なQM機能は不要 |
| Change | 中（見たい切り口が変わる可能性） | 柔軟に対応できる構造が望ましい |

**→ 疎結合（独立した小さなDB）で正解**
