**作成日**: 2026-03-11 Session 86

**前バージョン**: [session85/incoming-inspection-domain-model.md](../session85/incoming-inspection-domain-model.md)

## 1. 変更点（v1 → v2）

| 項目 | v1（Session 85） | v2（Session 86） |
|------|-----------------|-----------------|
| FW期待値 | ロットに固定 | **なし（全記録→後から比較）** |
| FW判定 | Pass/Fail | **記録のみ（多数派・はずれ値を可視化）** |
| 他項目の判定 | Pass/Fail | Pass/Fail（変更なし） |

---

## 2. 検査項目と判定方式

| 検査項目 | 期待値 | 判定方式 |
|----------|--------|----------|
| 通信疎通 | なし | 応答あり→Pass |
| シリアル番号 | なし | 取得成功→Pass |
| 出力レート | **ロット** | 期待値と比較→Pass/Fail |
| ポート設定 | **ロット** | 期待値と比較→Pass/Fail |
| **FWバージョン** | **なし** | **全記録、後から比較** |

---

## 3. ドメイン概念

### 3-1. エンティティ一覧

| エンティティ | 説明 | 主な属性 |
|-------------|------|---------|
| **ロット** | 同一仕様の装置群 | ロット番号、期待値（出力レート、ポート設定） |
| **検査** | 1回の検査作業 | 検査日時、対象装置シリアル、所属ロット |
| **検査項目結果** | 各項目の結果 | 項目、実測値、期待値、判定 |

### 3-2. 関係図

```
ロット
  │
  ├── 期待値（FW以外）
  │     - 出力レート（例: 100ms）
  │     - ポート設定（例: UBX+NMEA）
  │
  └── 検査 (1:N)
        │
        ├── 装置シリアル番号
        ├── 検査日時
        │
        └── 検査項目結果 (1:N)
              │
              ├── 項目（通信疎通/シリアル/FW/出力レート/ポート設定）
              ├── 実測値
              ├── 期待値（ロットから取得 or なし）
              └── 判定（Pass/Fail/Error/記録のみ）
```

### 3-3. 判定の種類

| 判定 | 意味 | 使用項目 |
|------|------|---------|
| Pass | 期待通り | 通信疎通、シリアル、出力レート、ポート設定 |
| Fail | 期待と異なる | 出力レート、ポート設定 |
| Error | 検査エラー（通信失敗等） | 全項目 |
| **Recorded** | 記録のみ（判定なし） | FWバージョン |

---

## 4. FWバージョンの特殊処理

### 4-1. なぜ特殊か

- FWは「期待値を事前に決められない」場合がある
- 同じロットでもFWが混在している可能性
- **複数台検査して多数派・はずれ値を見つける**のが目的

### 4-2. 運用フロー

```
1. 1台目を検査 → FW "HPG 1.32" を記録
2. 2台目を検査 → FW "HPG 1.32" を記録
3. 3台目を検査 → FW "HPG 1.30" を記録 ← はずれ値
4. ロット一覧で比較 → 多数派(1.32)とはずれ値(1.30)が見える
5. はずれ値の装置を別途確認
```

### 4-3. UI表示（ビューレベル）

ドメインモデルとしてはRecordedのまま保存。
表示時に集計して以下を算出:

- 多数派FW
- はずれ値FW（と該当装置）
- FW分布

## 6. 運用フロー

```
1. ロット登録（期待値を設定：出力レート、ポート設定）
     ↓
2. 装置をUSB接続
     ↓
3. ロットを選択して検査開始
     ↓
4. 各項目を実行
   - 通信疎通: 応答確認 → Pass/Error
   - シリアル: 取得 → Pass/Error
   - FW: 取得して記録 → Recorded
   - 出力レート: ロット期待値と比較 → Pass/Fail/Error
   - ポート設定: ロット期待値と比較 → Pass/Fail/Error
     ↓
5. 検査結果を保存
     ↓
6. ロット一覧でFW分布を確認（後から）
```
