### 3. Process（プロセス）

#### 3.1 検査工数の分布（時間がかかる作業の特定）

Excelの「検査工数(分)」列から、**工数の大きい検査**を特定できる:

**メカ（251件）の工数分布**（月別傾向あり: 最多2026-01=30件、最少2025-05=4件）:

| 工数範囲 | 該当部品例（Excelから） | 特徴 |
|---------|----------------------|------|
| 5分未満 | 小物部品（ナット、グロメット） | 員数確認が中心 |
| 5-15分 | 樹脂部品（吐出口、レール） | 寸法測定、外観検査 |
| 15-30分 | 複雑な樹脂部品 | 詳細な寸法測定、インサート確認 |
| 30分以上 | **バッテリー: 420分**（受入に時間がかかるものシート）<br>**H12 Custom: 250台を4名で131.5h**（買い物シート） | 特殊検査（充放電、電圧測定）<br>大規模検査 |

**記録期間と頻度**:
- メカ: 2024-09-03 〜 2026-02-19（約17ヶ月）、約14件/月
- エレキ: 2024-10-18 〜 2026-02-27（約16ヶ月）、約8件/月
- Api: 2024-11-08 〜 2026-03頃（約16ヶ月）、約6件/月
- **全体: 約567件 / 17ヶ月 = 約33件/月（1日1〜2件ペース）**

**ワークショップでの確認ポイント**:
- **Session 260で「梱包変更作業が大半」と分かった** → 検査工数の記録には梱包変更作業が含まれているか？
- 「検査工数15分」の内訳を聞く（開封、梱包変更、員数確認、外観検査、記録作成）
- 工数の変動（4〜30件/月）は生産計画に連動しているか

### 4. Outputs（出力）

#### 4.1 検査記録（Excel）

**記録されている情報**（メカシートのヘッダー・5行目）:
- 入荷日、検査完了日
- 発注番号（**メカのみ、エレキ・Apiにはない**）
- 品番、品名、サプライヤ
- 入荷数量、不良数量
- 検査内容、検査工数（分）、人工
- 検査場所、検査治具
- 検査結果・備考、品質基準書修正要否と箇所

**記録されていない情報**（Session 6で確認済み）:
- **ロット番号** → トレーサビリティに必要だが欠落
- **サプライヤーID** → 業者ごとの品質傾向分析に必要
- **発注番号**（エレキ・Api） → メカのみ記録、他は欠落

**列構成の不統一**:
- メカ: 発注番号あり（C列）、品番はD列
- エレキ・Api: 発注番号なし、品番はC列
- → DB化時に列のマッピングが必要

#### 4.2 合格/不合格の判定

Excelの「検査結果・備考」列から（1列に複数の情報が混在）:

| 結果 | 具体例（Excelから） | 備考 |
|------|------------------|------|
| 良（合格） | 大多数 | **しかし懸念事項が埋もれている**:<br>**行16**: 「良　ピン跡に黒ずみ（不問）」<br>**行60**: 「良　ノギスでの寸法計測に難しさがある」<br>**行63-67**: 「良　ノギスで測ると設計値とずれます」（5件連続） |
| 不良（不合格） | **行27**: 不良数=1「インサート逆入れあり、代品入庫済」<br>**行210**: 不良数=0「初回納入は、全数寸法NG。作り直して再納入分」 | 不良数量が記録されている<br>（ただし空欄も多い） |
| 判断保留 | **行130**: 不良数=「保留」<br>**Api行25**: 「穴が開いてないのが3つあり、処置後入荷、再発防止依頼要」 | 「不問にしたい」「要協議」「メーカーに確認要」など、対応ステータスが追跡されていない |

**課題**（Session 6 セクションF参照）:
- **1列に8種類の情報が混在**: 合否判定、不良の詳細、対処内容、測定結果、検査方法メモ、判断保留、基準書要望、作業備忘録
- 判定基準が属人的: **エレキ行142** 不良数=「12個くらい」「外装部打痕（個人主観）」

**ワークショップでの確認ポイント**:
- 合格/不合格の判定は誰がどのように行っているか
- 「不問」「不問にしたい」の判断基準は何か
- 判断保留になった場合、どう処理しているか（追跡方法はあるか）

## 工数の大きい作業の特定（Session 260 + Session 6）

### Session 260での発見

**梱包変更作業が作業時間の大半を占める**:

| 部品 | 入荷時の梱包 | 梱包変更作業の内容 | 推定工数 |
|------|------------|----------------|---------|
| parts1（小物部品） | ダンボール板に並べて梱包 | 小分けにしてビニール袋に「48pcs」等と書いて梱包し直し | 大（複数箇所） |
| parts2（小物部品） | 白い緩衝材で個別包装 | 緩衝材を外して箱に整然と並べ直し | 中（1箇所ずつ） |
| parts3（大物部品） | 発泡スチロール型枠にきれいに配置 | 梱包変更なし | 小 |

**仮説**: parts1とparts2のような梱包の場合、梱包変更作業に大きな工数がかかる

### Excelの「検査工数(分)」との照合

**確認すべきこと**:
1. Excelの「検査工数」に梱包変更作業は含まれているか？
2. 工数15分の検査のうち、何に何分かかっているか？
   - 開封: X分
   - 員数確認: Y分
   - 外観検査: Z分
   - **梱包変更作業**: W分 ← これが大きいはず
   - 記録作成: A分

## ワークショップで使う戦略

### 1. 事前に整理した情報を見せる

**効果**:
- ヒアリング時間を短縮（既に分かっている情報は確認のみ）
- 「業者ごとの傾向」「工数の内訳」など、Excelだけでは分からない部分に集中できる

**提示方法**:
- このドキュメント（[pre-extracted-info-from-excel.md](pre-extracted-info-from-excel.md)）をプリントアウト
- 「Excelからこういう情報が分かりました。ここを確認させてください」と示す

### 2. 工数の内訳を詳しく聞く

**質問例**（具体的な行番号を示す）:
- 「Excelのメカシート行27（吐出口、金森産業）の検査工数は何分ですか？」
- 「その15分の内訳を教えてください: 開封、員数確認、外観検査、梱包変更、記録作成、それぞれ何分くらいですか？」
- 「梱包変更作業は毎回発生しますか？部品によって違いますか？」

### 3. 写真と照合する

**質問例**（Session 260の写真を示しながら）:
- 「Session 260で撮影した parts1（グロメット・ナット）は、Excelのどのシート・行番号・品番ですか？」
- 「この部品の検査工数は何分ですか？そのうち梱包変更作業は何分ですか？」

### 4. 不良事例の詳細を聞く

**質問例**（具体的な行番号を示す）:
- 「行27の『インサート逆入れあり』はどのように発見しましたか？」
- 「行210の『初回納入は全数寸法NG』の場合、サプライヤーへの対応はどうしましたか？」
- 「行63-67で5件連続『ノギスで測ると設計値とずれます』とありますが、この問題はその後解決しましたか？」

## SIPOC作成時の活用方法

### Suppliers（供給者）

- [ ] Excelのサプライヤー一覧を見せて「この業者ごとに梱包方法が異なりますか？」と確認
- [ ] 不良が多い業者（金森産業、T-motor）は梱包も問題があるか確認

### Inputs（入力）

- [ ] Excelの品名一覧を見せて「部品カテゴリはこの分類で合っていますか？」と確認
- [ ] 検査資料（品質基準書、図面、納品書）の有無を確認

### Process（プロセス）

- [ ] 検査工数の内訳を詳しく聞く（開封、員数確認、外観検査、梱包変更、記録作成）
- [ ] 「検査工数」には何が含まれているか明確にする

### Outputs（出力）

- [ ] Excel記録の内容を確認（ロット番号等の欠落項目を指摘）
- [ ] 合格/不合格の判定基準を確認

### Customers（顧客）

- [ ] 推測したCustomersリストを見せて「誰が何を受け取っていますか？」と確認
