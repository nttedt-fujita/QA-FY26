## 詳細

### 1. SIPOCダイアグラムのテンプレート・ツール選定

**目的**: 田原さん・杉山さんとのワークショップで使用するテンプレートを用意

**選定基準**:
- 現場で使いやすいか（Excel or Draw.io）
- 編集・共有が容易か
- 視覚的に分かりやすいか

**検討事項**:
- Excel: 現場で馴染みがある、編集が簡単
- Draw.io: 視覚的に優れている、図の共有が容易

### 2. SIPOC作成の手順確認・チェックリスト作成

**SIPOC作成の目的（重要）**:
- ❌ 完璧な図を描くこと
- ✅ **現場と一緒に描きながら、分からないことを洗い出すこと**
- ✅ **現場と管理者の認識のズレを発見すること**

**SIPOC作成の流れ**:
1. Suppliers（供給者）の特定
2. Inputs（入力）の特定
3. Process（プロセス）の定義（5-7ステップ程度）
4. Outputs（出力）の特定
5. Customers（顧客）の特定

**作成中にやること**:
- 田原さん・杉山さんと一緒に図を描く（一方的に聞くのではなく、一緒に作る）
- 「これで合ってますか？」と確認しながら進める
- **追加ヒアリング項目をメモする**

**チェックリスト**:
- [ ] 部門横断チームの招集（田原さん、杉山さん、その他関係者）
- [ ] 各要素の洗い出し
- [ ] 非効率性・冗長性の発見
- [ ] 現場の声を反映
- [ ] **追加ヒアリング項目のメモ**（重要！）

### 3. ヒアリング項目の準備

**背景**:
- Session 260で田原さんヒアリング実施（現場写真6枚）
- 納品→入庫→出庫→組立→検査→出荷の全体フロー確認が必要

**Session 260で分かったこと**:
- 受入作業の実態（梱包変更作業の存在）
- 「なんとなく」で作業している状態
- 現場写真6枚（梱包状態）

**まだ分かっていないこと（たくさんある）**:

| カテゴリ | 未確認の内容 |
|---------|------------|
| **全体フロー** | 出庫、組立、検査、出荷の実態 |
| **詳細な時間** | 各作業の正確な所要時間 |
| **業者ごとの違い** | 業者Aは梱包OK、業者Bはバラバラ、みたいな傾向 |
| **問題の頻度** | 梱包変更作業は毎回？たまに？ |
| **他の担当者** | 杉山さん、他の現場の人の視点 |
| **記録の有無** | Excel以外に何か記録しているか |
| **トラブル事例** | 過去に困ったこと、ミスの事例 |

**戦略: SIPOC作成中に追加ヒアリング項目を洗い出す**:
- 事前に完璧なヒアリング項目リストを作るのは困難
- **SIPOC作成中に「ここ詳しく聞かないと」が見えてくる**
- 追加ヒアリング項目をメモする準備が重要

**SIPOC作成のための初期ヒアリング観点**:

| 観点 | 確認内容 |
|------|---------|
| Suppliers | どこから部品が届くか（国内/海外、業者名） |
| Inputs | 何が届くか（部品種類、梱包状態、書類） |
| Process | 受入作業の流れ（納品→入庫→出庫→組立→検査→出荷） |
| Outputs | 何が出力されるか（検査記録、入庫記録、出荷記録） |
| Customers | 誰に渡すか（組立担当、出荷担当） |

**SIPOC作成中にメモすべきこと**:
- 「ここ詳しく聞かないと」と思った箇所
- 田原さん・杉山さんが「うーん、たまにあるかも」と曖昧な返答をした箇所
- 現場と管理者の認識がズレている箇所

### 4. 暗黙知外部化の手法選定

**候補**:
1. 動画記録（スマホ・タブレット）
2. 手順書化（写真 + 説明文）
3. VRシミュレーション（コスト・時間がかかる）

**Session 260 の現場写真を活用**:
- 既に6枚の写真あり（梱包状態）
- 追加で作業中の写真・動画を撮影

**選定基準**:
- 現場の負担が少ない
- 暗黙知を捕捉できる
- 継続的に更新できる

### 5. 田原さん・杉山さんとの協議内容準備

**目的**: SIPOC作成ワークショップの日程調整・事前説明

**協議内容**:
- SIPOC とは何か（簡単な説明）
- なぜ作るのか（目的・メリット）
- どのくらい時間がかかるか（1-2時間程度）
- 誰が参加すべきか（受入担当者、組立担当者等）
