# 課題分析（As-Is）

**作成**: Session 317 (2026-03-26)
**使用フレームワーク**: MECE、ロジックツリー（Whatツリー）
**目的**: 品質管理プロセス全体の課題を網羅的に把握する

---

## 1. MECE分解: 品質管理プロセスの課題

### 切り口: 品質管理プロセスの工程（時系列）

品質管理の工程フロー全体をMECEに分解する。

```
品質管理プロセスの課題
├─ 1. サプライヤ管理
├─ 2. IQC（受入検査）      ← M3
├─ 3. 製造工程管理
├─ 4. IPQC/PQC（工程内検査）← M4
├─ 5. FQC（最終検査）
├─ 6. OQC（出荷検査）
├─ 7. 市場品質（CS対応）
└─ 8. 横断: データ連携・集計
```

**MECEチェック**:
- モレ: FQC/OQCを含めた（存在不明でも工程として漏らさない）
- ダブり: 各工程は時系列で排他的

## 2. ロジックツリー: 各工程の課題（Whatツリー）

### 1. サプライヤ管理

```
サプライヤ管理の課題
├─ サプライヤ名の表記揺れが多数 [A]
│    └─ 「金森産業（株）」vs「金森産業株式会社」等、名寄せが必要 [A]
├─ 品質協定書が未締結 [A]
│    └─ 委託先からの工程データ取得に制約 [A]
└─ サプライヤ別品質傾向の分析が困難 [A]
     └─ 表記揺れにより正確な集計ができない [A]
```

### 2. IQC（受入検査）= M3

```
受入検査の課題
├─ 記録の不完全性
│    ├─ 杉山さん分のみデジタル。他は紙 [A+B]
│    ├─ 入荷日欠損34件 [A]
│    ├─ 工数未記入約80件 [A]
│    └─ 未来日付10件（入力ミス）[A]
├─ データ構造の問題
│    ├─ 品名の表記揺れ（約220種）[A]
│    ├─ 検査結果と備考が1列に混在 [A]
│    ├─ 検査数量に87パターンの表記 [A]
│    └─ メカ/エレキ/Apiで列構成が異なる [A]
├─ 概念の欠落
│    ├─ ロット概念がない [A]
│    ├─ サプライヤIDがない [A]
│    ├─ 発注番号がない [A]
│    └─ 検査基準（AQL等）がない [C]
├─ kintoneとの断絶
│    ├─ 受入検査ExcelとkintoneにS/N等の共通キーなし [A]
│    └─ kintone「受入製品」アプリとの関係が不明 [D]
└─ 工程不良との断絶
     └─ M3→M4の因果追跡不可 [A]
```

### 3. 製造工程管理

```
製造工程管理の課題
├─ 製造工程の詳細が不明 [C]
│    └─ マニュアルに「製造登録」のみ [A(M)]
├─ ナカヨ・ワタナベの工程情報が取得できていない [A]
│    └─ 品質協定書未締結のため [A]
└─ GENの製造登録の実態が不明 [A(M)]
     └─ マニュアル記載のみ。実務者確認なし
```

### 4. IPQC/PQC（工程内検査）= M4

```
工程内検査の課題
├─ デジタル管理されていない [A]（藤田さん確認）
├─ 記録方法自体が不明 [D]
│    ├─ 紙？口頭？写真？
│    ├─ 記録者は誰？
│    └─ 記録頻度は？
├─ 不良の定義・分類が不明 [D]
│    └─ コード体系がない
├─ 受入検査との紐づきがない [A]
└─ CS問合せとの紐づきがない [A]
     └─ kintoneに「製造ミス」原因分類があるが元記録がない [A]
```

### 5. FQC（最終検査）

```
最終検査の課題
├─ 存在自体が不明 [D]
├─ kintone機体マスタに「検収担当者」あり [A]
│    └─ 検査なのか納品確認なのか不明 [D]
└─ 品質記録がない（あったとしても）[D]
```

### 6. OQC（出荷検査）

```
出荷検査の課題
├─ 存在自体が不明 [D]
├─ kintone発送(08)に「検品」ステップあり [A]
│    └─ 品質検査かピッキング確認か不明 [D]
└─ 品質記録がない（あったとしても）[D]
```

### 7. 市場品質（CS対応）

```
市場品質の課題
├─ kintone農業問合せ(19)で管理されている [A]
├─ 22種の原因分類がある [A]
│    ├─ 「製造ミス（組立工程）」[A]
│    ├─ 「検査漏れ（受入検査/飛行検査後）」[A]
│    └─ 原因分類は工程不良ではなく市場不具合 [A]
├─ 工程記録との紐づきがない [A]
│    └─ 「製造ミス」を選んでも元の工程記録を参照できない
└─ 仕損品管理が3アプリに分散 [A]
     └─ 統合管理されているか不明 [D]
```

### 8. 横断: データ連携・集計

```
データ連携の課題
├─ kintone ↔ GEN
│    ├─ kintone→GEN: CSVエクスポート→インポート（一部マスタ）[A]
│    ├─ GEN→kintone: 経理マニュアル記載あり [A(M)] 要再確認
│    └─ 連携の実態が不透明 [A(M)]
├─ kintone/GEN ↔ PSI Excel
│    ├─ 全て手動統合 [A]
│    ├─ 掛川さん週2h・年104h [A+B]
│    └─ データソースの全リスト未確認 [B]
├─ 受入検査Excel ↔ 全て
│    ├─ kintoneとの紐づきなし [A]
│    ├─ GENとの紐づきなし [A]
│    └─ 完全に孤立したデータ [A]
└─ 群馬通商Excel ↔ kintone
     ├─ 両方にロケーション情報あり [A]
     └─ 同期方法・タイミングが不明 [B]
```

---

**確度凡例**: A=事実確認済み / A(M)=マニュアル記載（実態要確認）/ B=複数情報源から推定 / C=単一情報源 / D=未確認
