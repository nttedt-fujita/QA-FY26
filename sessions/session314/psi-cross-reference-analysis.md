## 2. 群馬通商Excel → PSI_Mar23 出庫行の突合

### 群馬通商 月別出庫集計（FY25）

| 品目 | 群馬通商での状況 |
|------|-----------------|
| 機体 | FY25の出庫データがほぼなし（2025年1-3月に集中、4月以降は極少） |
| バッテリ | 4-7月に出庫あり、8月以降は入庫のみ |
| チャージャー | 全月で出庫あり |
| 散布装置 | 4-7月に出庫あり、8月以降は入庫のみ |

### 考察

群馬通商の出庫データとPSI出庫行の突合は**直接的な一致が見られない**。理由として:

1. **群馬通商は在庫保管の委託先**であり、出庫=群馬通商からの発送
2. PSIの「出庫実績」は**群馬通商からの出庫だけでなく、他の出庫ルートも含む**可能性
3. 群馬通商Excelの出庫日付は**実際の発送日**であり、PSIの月次集計とタイミングがずれる可能性

→ 群馬通商Excelは「在庫(NTT)」「在庫(SL)」の在庫数把握に使っている可能性が高い（出庫行より在庫行のソースとして機能）

## 3. コメント分析

PSI_Mar23のセルコメント（282件）の内容パターン:

| パターン | 例 | 確認済み事実 [A] | 推定ソース | 確度 |
|---------|-----|-----------------|-----------|------|
| 顧客別内訳 | 「中国エア 1, 鈴八 1, 富士電子 1」 | コメント内の顧客名がAgriシートの顧客名と一致 | kintoneの情報経由 | **B** |
| 倉庫在庫 | 「群馬倉庫 ペラ無し：70 現状：15」 | 「群馬倉庫」の記述がコメントに存在 | 群馬通商Excel | **C** |
| 保管場所 | 「志木倉庫 ペラ無し：76」 | 「志木倉庫」の記述がコメントに存在 | 不明 | **D** |
| 確保フラグ | 「内20台は『確保』フラグ付き」 | コメント内容を確認 | 手動入力 | **C** |
| 対応メモ | 「20台 代替機対応」 | コメント内容を確認 | 手動入力 | **C** |

→ **事実 [A]**: コメントの顧客別内訳の顧客名は、受注案件一覧_Agriの顧客名と一致している。
→ **推定 [B]**: Agriの情報（元はkintone [B]）を見ながらコメントに内訳を手入力している可能性が高い。kintoneの画面を直接見ている可能性もある。

## 5. プロトタイプに必要なこと

### 確度Aで言えること

Agriシートの数値からPSI受注行は再現できる。月別自動集計ができればPSI受注行の更新は自動化可能。

### 確度Bの前提（掛川さんへの確認が必要）

「kintoneの情報だけで自動化できる」は、kintoneがAgriの元データであることが前提。掛川さんへの確認事項:
- Agriシートの情報源は何か（kintone CSV? kintone画面? その他?）
- 情報の取得方法（CSV出力? 画面を見て手入力?）

### 最小構成（前提が確認できた場合）

1. **受注データ取り込み** → Agri相当のデータをDBに格納
2. **月別自動集計** → 品目別×月別の受注数を自動算出
3. **在庫計算** → 入庫 - 出庫（受注）の自動計算
