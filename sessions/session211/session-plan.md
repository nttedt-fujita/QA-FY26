# Session 211 計画

**目的**: 古い機の定期出力問題の根本対処を検討

**前提**: Session 209-210で個別メッセージ追加を試みたが、際限なく出てくる

---

## やること

| # | 作業 | 読むべきファイル |
|---|------|-----------------|
| 1 | CFG-CFG（設定リセット）の仕様を確認 | PDF p.63付近 |
| 2 | 設定リセットAPIの実装検討 | - |
| 3 | （代替案）全NAVメッセージ一括無効化の検討 | - |

---

## 背景

古い機（RTK基準局設定）には多数のメッセージが有効化されている。
個別にKey IDを追加する方式では：
- Session 199-201: 24キー
- Session 209: 34キー
- Session 210: 50キー

…と際限なく増え続ける。

---

## 検討すべき根本対処

1. **CFG-CFG（設定リセット）**
   - BBRをクリアしてデフォルト設定に戻す
   - その後、必要な設定のみ再投入

2. **全NAVメッセージ一括無効化**
   - NAV Class (0x01) の全メッセージをループで無効化
   - CFG-MSGOUT-UBX_NAV_*_USB/UART1 を全て0に

3. **古い機専用の初期化フロー**
   - 検出時にFWバージョンを確認
   - 古い機の場合は設定リセットを実行
