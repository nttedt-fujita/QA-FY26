## 3. M3：受入検査DB化

### ロット管理について

| 事実 | 内容 |
|------|------|
| 現状 | ロット管理ができていない |
| 原因 | 中国サプライヤはロット自体が無いことが多い、あっても信用できない |
| 方針 | **ロット定義はこちら（品質G）で決めてしまって良い**（石川さん承認） |
| 具体的な手段例 | 受入日・受入番号等をロットとして自社で定義する |

### 現行Excelの問題

- 「Excelを見ても中身がわからない」状態
- 過去セッション（Session 6）のExcelレビューと一致：構造化されていないためデータとして使えない

### バッテリーの検査体制の問題

| 観点 | 現状 |
|------|------|
| 外観検査 | SCMが実施 |
| 中身の検査（Log解析等） | 石川さんが実施 |
| 不合格時の記録・反映 | **できていない** |
| 検査データの管理者 | **決まっていない** |

### 履歴管理・スケジューリングの問題

- いつ何が何個入ってきているか分からない
- → **スケジュールが組めない**
- → **入荷履歴の追跡ができない**
