## ワークショップの流れ（詳細タイムライン）

### Phase 0: 導入（5-10分）

**目的**: SIPOCの説明、ワークショップの目的共有

**実施内容**:
1. 自己紹介・役割確認
2. SIPOC手法の簡単な説明
   - 5要素（Suppliers, Inputs, Process, Outputs, Customers）の説明
   - 高レベル（30,000フィート）の俯瞰図であることを強調
   - 詳細プロセスマップ（VSM）との違い
3. 今日の目的
   - 「完璧な図を描く」ことではなく「分からないことを洗い出す」こと
   - 追加ヒアリング項目を見つけることが重要

**使用資料**:
- sipoc-workshop-hearing-items.md（ワークショップの流れセクション）

---

### Phase 1: Process（プロセス）の定義（15-20分）

**目的**: 受入検査プロセスの高レベルステップを4-7個に整理

**実施方針**:
- Session 260で分かったフロー「入荷 → 開封 → 受入検査 → 梱包変更作業 → 入庫 → 出庫」を叩き台として提示
- 田原さん・杉山さんに「この流れで合っていますか？」と確認
- **詳細すぎないように注意**（4-7ステップに抑える）

**確認すべきポイント**:
| # | 確認項目 | 質問例 |
|---|---------|--------|
| P1 | プロセスの開始点 | 「入荷の連絡はどう来ますか？（メール、電話、システム）」 |
| P2 | プロセスの終了点 | 「入庫完了の記録はどうしていますか？」 |
| P3 | 検査のタイミング | 「員数確認は開封前ですか？開封後ですか？」 |
| P4 | 梱包変更作業の位置づけ | 「梱包変更作業は受入検査の一部ですか？それとも別の工程ですか？」 |

**出力（付箋）**:
- Process列に4-7個のステップを付箋で貼る

---

### Phase 2: Suppliers（供給者）と Inputs（入力）の特定（15-20分）

**目的**: プロセスに入ってくる「誰から」「何が」を明確にする

**実施方針**:
- pre-extracted-info-from-excel.mdの「Suppliers」「Inputs」セクションを見せる
- 「Excelからこういう情報が分かりました。これで合っていますか？」と確認
- 追加で出てくるSuppliersやInputsをメモ

**Suppliers確認ポイント**:
| # | 確認項目 | 質問例 |
|---|---------|--------|
| S1 | サプライヤー一覧 | 「主要なサプライヤーは何社くらいですか？」 |
| S2 | 業者ごとの梱包方法 | 「金森産業とT-motorでは梱包方法が違いますか？」 |
| S3 | 物流業者 | 「配送は誰がしていますか？（ヤマト、佐川等）」 |

**Inputs確認ポイント**:
| # | 確認項目 | 質問例 |
|---|---------|--------|
| I1 | 部品の種類 | 「部品は全体で何種類くらいありますか？」 |
| I2 | 梱包材 | 「どんな梱包材がよく使われますか？」（写真を見せながら） |
| I3 | 納品書・書類 | 「納品書、検査基準書はどのタイミングで受け取りますか？」 |
| I4 | ロット情報 | 「ロット番号、製造日などの記載はありますか？」 |

**出力（付箋）**:
- Suppliers列にサプライヤー名を付箋で貼る
- Inputs列に入力要素を付箋で貼る

---

### Phase 3: Outputs（出力）の定義（10-15分）

**目的**: プロセスから出てくる成果物を明確にする

**実施方針**:
- pre-extracted-info-from-excel.mdの「Outputs」セクションを参照
- 「検査記録（Excel）」「合格ロット」「不合格報告書」等を確認

**確認ポイント**:
| # | 確認項目 | 質問例 |
|---|---------|--------|
| O1 | 検査記録 | 「Excelのどのシートに記録していますか？記録タイミングは？」 |
| O2 | 合格ロット | 「入庫指示はどの形式で誰に出しますか？」 |
| O3 | 不合格報告書 | 「不良品が見つかったらどう記録しますか？」 |
| O4 | 梱包変更後の部品 | 「全ての部品が小分けにされるわけではないですよね？」 |

**出力（付箋）**:
- Outputs列に出力物を付箋で貼る

---

### Phase 4: Customers（顧客）の特定（10-15分）

**目的**: プロセスの出力を受け取る人・部門を明確にする

**実施方針**:
- pre-extracted-info-from-excel.mdの「Customers」セクションを参照
- 推測したCustomersリストを見せて「誰が何を受け取っていますか？」と確認

**確認ポイント**:
| # | 確認項目 | 質問例 |
|---|---------|--------|
| C1 | 組立担当 | 「合格ロットは誰に渡しますか？組立業者ですか？」 |
| C2 | 倉庫担当 | 「入庫作業は誰が行いますか？」 |
| C3 | 品質管理担当 | 「検査記録は誰が見ますか？」 |
| C4 | サプライヤー | 「不合格品が見つかったら、サプライヤーにどう連絡しますか？」 |

**出力（付箋）**:
- Customers列に顧客を付箋で貼る

---

### Phase 5: 全体レビューと追加項目の洗い出し（15-20分）

**目的**: SIPOC全体を見直し、抜け漏れや認識のズレを発見

**実施内容**:
1. 完成したSIPOC図を全員で眺める
2. 「この流れで合っていますか？」と確認
3. **追加ヒアリング項目をメモ**
   - 「ここ詳しく聞かないと」という箇所
   - 認識のズレが発見された箇所
   - 曖昧な返答があった箇所

**メモすべき内容**:
| カテゴリ | メモ項目 |
|---------|---------|
| 追加ヒアリング項目 | 「〇〇の詳細を次回確認」 |
| 認識のズレ | 「Excelでは××だが、実際は△△」 |
| 曖昧な返答 | 「〇〇は『たぶん』という回答だった」 |
| 測定可能性 | 「この工程の工数を測定できていない」 |

**出力**:
- 追加ヒアリング項目リスト（ノートにメモ）
- SIPOC図の写真撮影

---

### Phase 6: クロージング（5分）

**実施内容**:
1. 今日の成果物確認（SIPOC図、追加ヒアリング項目）
2. 次のステップの説明
   - Draw.ioでデジタル化
   - 追加ヒアリング項目の整理・優先順位付け
   - VSM（詳細プロセスマップ）作成の計画
3. お礼

---

## ワークショップ後のアクション（Session 263終了前に実施）

### Action 1: SIPOCのデジタル化

**実施内容**:
- ホワイトボードの写真をもとにDraw.ioで清書
- sipoc-iqc-template.drawioをベースに作成
- ファイル名: `sipoc-iqc-as-is.drawio`（現状のSIPOC）

**配置場所**:
- `sessions/session263/sipoc-iqc-as-is.drawio`

---

### Action 2: 追加ヒアリング項目の整理

**実施内容**:
- ワークショップで出た追加ヒアリング項目を整理
- 優先順位付け（P0-P3）
- Markdownファイルにまとめる

**ファイル名**: `additional-hearing-items.md`

**フォーマット**:
```markdown
# 追加ヒアリング項目（SIPOC作成後）

| 優先度 | カテゴリ | 項目 | 確認方法 |
|--------|---------|------|---------|
| P0 | Process | 梱包変更作業の詳細手順 | 現場観察 |
| P1 | Inputs | ロット番号の記載有無 | 実物確認 |
...
```

---

### Action 3: VSM作成の計画

**実施内容**:
- SIPOCで高レベルのプロセスが見えた
- 次はVSM（詳細プロセスマップ）で工数・ムダを測定
- VSM作成の計画を立てる

**確認すべきこと**:
- どのプロセスステップに時間がかかっているか
- どこにムダ（待機、手戻り、重複作業）があるか
- 梱包変更作業の詳細ステップ

**ファイル名**: `vsm-creation-plan.md`
