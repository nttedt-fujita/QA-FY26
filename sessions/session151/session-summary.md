# Session 151 サマリー

**日時**: 2026-03-12
**目的**: 定期出力無効化が効いていない理由を調査

---

## 実施内容

### 1. 「不明データ」の正体を特定

ログ分析により、「不明データ」の正体を特定:
- バイト値が0x20〜0xA0前後（dBHz値のパターン）
- **MON-SPANのスペクトラムデータ（定期出力）** が混入していた

### 2. 根本原因を特定

**問題**: `Layer::Bbr` のみでは即座に有効にならない

u-blox仕様:
- **RAM (0x01)**: 即座に有効。電源OFFで消える
- **BBR (0x02)**: 電源再投入時に有効。バッテリバックアップで保持

Session 149で `Layer::Bbr` に変更したが、これは**次回起動時から**有効になる設定だった。

### 3. 修正

`Layer::RamAndBbr` (0x03) を追加し、RAM + BBR両方に書き込むように変更。

**変更ファイル**:
- `cfg_valset.rs`: `Layer::RamAndBbr = 0x03` を追加
- `device_api.rs`: `disable_periodic_output(Layer::RamAndBbr)` に変更

### 4. 実機検証

**結果**: 一定の効果あり
- 修正前: 「不明データあり」が大量に発生
- 修正後: 「不明データあり」が0件

**しかし**: タイムアウト・パースエラーは依然発生（完全解決ではない）

---

2. **CFG-VALGETの実装**
   - 今回は不要だったが、将来的に設定読み出しが必要になる可能性
