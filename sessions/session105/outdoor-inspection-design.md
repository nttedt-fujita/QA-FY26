## 屋外テストで確認すべきこと

### 1. 受信感度（C/N0）

| 項目 | 内容 |
|------|------|
| 衛星ごとのC/N0 | dBHz単位。30〜50dBHzが正常範囲 |
| L1/L2別の感度 | L2は低仰角で欠損しやすい |
| 仰角別の傾向 | 15度以下は受信不可が正常 |
| Ref機との差 | 通常5〜15dBHz低下が許容範囲 |

### 2. 衛星情報

| 項目 | 内容 |
|------|------|
| 捕捉衛星数 | 多コンステレーションで30本以上 |
| 仰角（elevation） | 0〜90度 |
| 方位角（azimuth） | 0〜360度 |
| 信号品質 | qualityInd（1〜7） |

### 3. 合格基準の候補

[02-reception-sensitivity.md](../../docs/missions/m1-sensor-evaluation/gnss/02-reception-sensitivity.md) より:

| 観点 | 検討方向 |
|------|---------|
| L1受信感度 | 絶対値の下限（例: 24dBHz以上） |
| QZSS L1 | 最低値で足切り（No.5の13dBHzは異常） |
| 仰角マスク | 15度以上の衛星のみ評価 |

## 必要なUBXメッセージ

| メッセージ | Class/ID | 用途 |
|------------|----------|------|
| **NAV-SAT** | 0x01 0x35 | 衛星情報（cno, elev, azim, qualityInd） |
| NAV-SIG | 0x01 0x43 | 信号情報（L1/L2別のcno） |
| NAV-PVT | 0x01 0x07 | 測位結果（numSV, fixType） ※既存 |

### NAV-SATの出力内容

| フィールド | 型 | 説明 |
|------------|-----|------|
| gnssId | U1 | GNSS種別（0=GPS, 1=SBAS, 2=Galileo, 3=BeiDou, 5=QZSS, 6=GLONASS） |
| svId | U1 | 衛星番号 |
| cno | U1 | C/N0 (dBHz) |
| elev | I1 | 仰角 (度) |
| azim | I2 | 方位角 (度) |
| flags | X4 | 品質・使用状況フラグ |

### NAV-SIG vs NAV-SAT

| 項目 | NAV-SAT | NAV-SIG |
|------|---------|---------|
| 粒度 | 衛星単位 | 信号単位（L1/L2/L5別） |
| C/N0 | 衛星の代表値 | 信号ごとの値 |
| 用途 | スカイプロット | L1/L2詳細比較 |

**方針**: まずはNAV-SATを実装。L1/L2詳細が必要ならNAV-SIGを追加。

## UI要件

### 1. スカイプロット（天球図）

```
        N (0°)
         │
    ┌────┼────┐
    │    │    │
W ──┼────┼────┼── E
    │    │    │
    └────┼────┘
         │
        S (180°)
```

- 極座標表示（中心=天頂、外周=水平線）
- 仰角: 中心=90°、外周=0°
- 方位角: 上=北(0°)、右=東(90°)
- 衛星をドットで表示
- C/N0を色で表現（緑=高、黄=中、赤=低）
- コンステレーション別の形状（●=GPS, ■=QZSS, ▲=GLONASS, ◆=Galileo）

### 2. C/N0棒グラフ

```
G01 ████████████████████ 42 dBHz
G02 ██████████████████████████ 48 dBHz
G07 ████████████ 29 dBHz
...
Q02 ██████████████████ 38 dBHz
```

- 衛星ごとの横棒グラフ
- L1/L2を別の色（L1=緑、L2=青）
- コンステレーション別にグループ化
- 閾値ライン表示（例: 30dBHz）

### 3. 更新頻度

- 1秒ごとの更新（NAV-SATの出力レートに依存）
- リアルタイム性よりも安定性を優先

## 技術的な考慮事項

### 1. ポーリング vs 自動出力

| 方式 | メリット | デメリット |
|------|---------|-----------|
| ポーリング | 必要な時だけ取得 | レイテンシ |
| 自動出力（CFG-MSG） | 低レイテンシ | バッファ管理が複雑 |

**方針**: 屋外検査中は自動出力（1秒間隔）、屋内検査中はOFF

### 2. フロントエンド表示

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| Canvas（JavaScript） | 自由度高い | 実装コスト |
| SVG | 拡大縮小に強い | 動的更新が重い |
| Chart.js等ライブラリ | 実装が楽 | カスタマイズ制限 |

**方針**: スカイプロットはCanvas、棒グラフはChart.js
