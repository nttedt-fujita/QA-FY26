## 2. ミッション優先順位

### 推奨: M3/M4先行 → M1/M2並行

| 優先度 | ミッション | 理由 |
|--------|-----------|------|
| **1位** | M3/M4（受入検査DB・工程不良DB） | ソフトスキルが直接活きる、早期成果が出やすい |
| **2位** | M1（センサー評価手法） | 1Q開始だが、Step1は調査中心で並行可能 |
| **3位** | M2（点群データ検証） | M1の成果が入力になるため、2Q以降に本格着手 |

### M3/M4を先にすべき5つの理由

1. **ソフトスキルが最も直接的に活きる** — DB設計、SQL、データ分析
2. **早期に成果が出せる** — DBが動けば即効果
3. **他メンバーの業務を支援できる** — 品質G全員が使う
4. **M3/M4は一体設計すべき** — lot_idで紐付けて相関分析可能
5. **M1 Step1は座学・調査中心** — 並行で軽めに進められる

## 3. M1: AirGrow搭載センサーの定量的評価手法策定

### 対象センサー
- **M1-A: Lidar** — 測距精度、点群密度等
- **M1-B: GNSS** — RTK Fix精度、Fix率等

### 【NEW】現状認識（Session 11-12で判明）

| サブミッション | 現状 | 課題 |
|---------------|------|------|
| **M1-A: Lidar** | 評価環境ゼロ。規格・設備をゼロから探す必要あり | 設備投資 or 外部委託の判断が必要 |
| **M1-B: GNSS** | 小板橋さんが既存知見を持っている | ヒアリングで現状把握が先 |

**重要な発見**: M1は2つのセンサー（Lidar/GNSS）を抱えており、それぞれ状況が全く異なる。

### 【NEW】M1-A（Lidar）の進め方

#### 現状
- 評価環境: **室内・屋外ともに存在しない**
- センサー: 開発中の製品に載せる予定（まだ決まっただけ）
- 過去実績: センサー評価・検証は**全て初めて**

#### 関連規格

| 規格 | 内容 | 状況 |
|------|------|------|
| ASTM E2938-15 | 中距離3Dイメージングの測距性能評価 | 発行済み |
| ASTM E3125-17 | 車載LiDARの性能評価 | 発行済み |
| ISO/CD 13228 | 自動車用LiDARの試験方法 | 策定中 |
| ISO/TS 19159-2 | 航空LiDARセンサーの校正 | 発行済み |

#### 評価環境構築の選択肢

| オプション | 概要 | メリット | デメリット |

### Step 1（1Q）の具体的作業

```
Step 1: 評価手法の調査・計画策定
  ├── [GNSS] 小板橋さんヒアリング → 既存知見の整理・標準化
  ├── [Lidar] 外部調査 → 評価環境構築の選択肢検討
  ├── 評価環境（室内/屋外）の設計
  └── 【成果物】評価計画書
```

### Lidar評価の主要指標（7項目）

| 指標 | 定義 | 典型値 |
|------|------|--------|
| 測距精度 | 真値に対するバイアス | 1〜3 cm @ 100m |
| 測距精密度 | 繰り返し測定のばらつき | 0.5〜2 cm |
| 角度精度 | スキャンミラー角度誤差 | 0.01°〜0.05° |
| レンジノイズ | 短時間測距値の変動 | 反射率・距離依存 |
| 最大検出距離 | 検出可能な最大距離 | 100〜450m |
| 点群密度 | 単位面積当たりの点数 | 50〜400 pts/m² |
| マルチエコー分離能 | 複数反射を分離できる最小距離差 | 0.5〜1.5m |

### GNSS評価（RTK-GNSS）の主要項目（5項目）

| 項目 | 基準例 |
|------|--------|
| Fix率 | > 95% (開放環境) |
| TTFF | < 30秒 |
| Fix精度（水平） | < 2cm + 1ppm × 基線長 |
| Fix精度（垂直） | < 3cm + 1ppm × 基線長 |
| Wrong Fix率 | < 0.1% |

## 4. M2: 点群データ検証方法の策定

### 概要
AirGrowが取得する点群データの品質を検証する方法を策定。

### 【NEW】現状認識（Session 11-12で判明）

| 項目 | 内容 |
|------|------|
| Lidarの用途 | みかん畑の上を飛行、木を検知して避ける |
| 飛行方式 | 2D飛行経路を事前設定 + Lidarでリアルタイム障害物検知 |
| 現在の課題 | **False Alarm（誤検知）** — 空間に存在しないものを存在すると検知 |
| 作業状況 | False Alarm低減作業を**別プロジェクトで実施中** |
| 前例 | なし。開発中で初めての取り組み |

### 【NEW】M2の進め方

```
別プロジェクトで進行中のFalse Alarm作業
        ↓
   作業内容を整理・文書化
        ↓
   M2の検証手法として体系化
```

**ポイント**: 現在進行中の実務作業をベースに検証手法を構築する（机上の空論ではなく実践から）

### スケジュール

| フェーズ | 時期 | 内容 |
|---------|------|------|
| **Step 1** | 2Q | 品質指標の定義と検証手順の策定 |
| **Step 2** | 2Q-3Q | 検証パイプラインの構築 |
| **Step 3** | 3Q-4Q | 試行運用と改善 |

### M1との依存関係

- **M2はM1の後に本格着手**
- M1の成果（評価指標）がM2の入力になる
- 1QはM1の調査フェーズと並行で指標の定義を検討可能
- **False Alarm作業の整理は並行で進められる**

### 品質指標（7項目）

| 指標 | 定義 | 目標値例 |
|------|------|---------|
| 点密度 | 単位面積当たりの点数 | ≥ 8 pts/m² |
| 点密度の均一性 | 変動係数 | CV < 30% |
| ノイズ率 | 外れ値の割合 | < 0.5% |
| 絶対精度 | GCPに対するRMSE | RMSE_z < 10cm |
| 相対精度 | ストリップ間の整合性 | < 7cm |
| 網羅性 | カバー率 | ボイド < 4 × NPS² |
| 分類精度 | 点群分類の正答率 | 地表面分類 > 95% |

### 検証ツール

| ツール | 用途 | ライセンス |
|-------|------|-----------|
| CloudCompare | 点群比較・解析 | OSS |
| PDAL | 点群処理パイプライン | OSS |
| LAStools | LASファイル処理 | 一部商用 |
| RTKLIB | GNSS後処理解析 | OSS |
