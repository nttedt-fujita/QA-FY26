# FY26 在庫管理DXロードマップ

## 概要

調達→組立→出荷を一貫管理する「統合在庫管理システム」を目指し、FY26は4つのフェーズで段階的に進める。

**ミッションの位置づけ**:
- M3（受入検査DB）/ M4（工程不良DB）→ **本ロードマップ（DX施策）に統合**
- M1（GNSS評価）→ 4-6月に受け入れ検査、並行して進行
- M2（障害物検知評価）→ 開発側との連携待ち、状況を見て判断

---

## kintone契約方式

### 現状

kintoneはNTT東日本の基盤上で利用中。
→ API連携にはNTT東日本の許可が必要

### 課題

統合在庫管理システムの実現には**API連携が必須**:
- GEN/kintoneからの自動データ取得
- リアルタイムの在庫可視化
- 将来的なAI連携

### 対応方針

2つの選択肢:
1. NTT東日本の許可を取得する
2. NTTeDT独自でkintone契約する

→ **許可取得の手続きを考えると、独自契約のほうが早い**

### 進捗

和田さん・宮崎さんが対応中

---

## 将来ビジョン: AI連携

統合在庫管理システムが整った先には、AI連携による高度化が可能になる。

### AI連携でできること（例）

| AI活用 | 内容 | 必要なデータ |
|--------|------|-------------|
| **需要予測** | 過去のPSIから将来の需要を予測 | PSI履歴 |
| **発注最適化** | リードタイム+需要予測で最適発注点 | 在庫データ+リードタイム |
| **欠品リスクアラート** | 危険水準で自動通知 | リアルタイム在庫 |
| **異常検知** | 通常と異なる在庫変動を検出 | 在庫推移データ |

### FY26での着手候補

**欠品リスクアラート**が最も現実的:
- 閾値ベースなら実装難易度が低い
- Phase 4の延長として組み込み可能
- 小笠原さんの課題（実在庫が見えない）の解決に直結

→ Phase 1-4の土台ができた上で、年度末に着手を検討

- 部品在庫（入庫/出庫/残数/ロット・シリアル）

- 製品在庫（組立機体/出荷準備）

### Phase 1: 集計業務の自動化

| 項目 | 内容 |
|------|------|
| **期間** | 4月〜6月（M1 GNSSと並行） |
| **目的** | 掛川さんのサービス残業解消 |
| **解決する課題** | PSI集計の手作業（週2h → 自動化） |
| **やること** | GEN/kintoneからデータ取得 → PSI形式で自動集計 |
| **成果物** | PSI自動集計ツール |
| **副次効果** | GEN/kintoneのデータ構造を深く理解できる |

**前提条件**:
- GEN/kintoneからのデータエクスポート方法の確認
- （API不可の場合）CSV手動エクスポートで対応

**備考**: 4月は半分以上がM1 GNSS作業のため、並行作業として進行

---

### Phase 2: 入出庫の自動記録

| 項目 | 内容 |
|------|------|
| **期間** | 7月〜9月（Phase 1完了後、またはPhase 3と並行） |
| **目的** | 受入検査・棚の入出庫をデジタル化 |
| **解決する課題** | 手書き・二重入力、履歴が追えない |
| **やること** | 受入検査時・棚の入出庫時にQRコードでスキャン記録 |
| **成果物** | QRコード入出庫管理システム |
| **副次効果** | ロット・シリアルのトレーサビリティ確保 |

**前提条件**:
- 棚・ロットへのQRコード貼付運用の合意
- スキャンデバイス（スマホ/ハンディ）の選定

**参考図**:
- [受入検査フロー図](../session297/iqc-flow-current-and-ideal.drawio)
- [QRコード入出庫管理図](../session297/qr-inventory-management.drawio)

---

### Phase 3: 委託先在庫の見える化

| 項目 | 内容 |
|------|------|
| **期間** | 7月〜12月 |
| **目的** | 小笠原さんの課題解決 |
| **解決する課題** | 製造委託先の実在庫が見えない |
| **やること** | 製造委託先からの在庫報告を受け取り可視化 |
| **成果物** | 製造委託先実在庫ダッシュボード |
| **副次効果** | 欠品予測の精度向上、発注判断の改善 |

**前提条件**:
- 製造委託先との連携方法の合意（Excel/フォーム/API）
- 報告頻度の決定（日次/週次）

**検討事項**:
- 製造委託先が複数ある場合のフォーマット統一
- 群馬通商の在庫管理ツールとの連携

---

### Phase 4: SCM全体の一元管理

| 項目 | 内容 |
|------|------|
| **期間** | 1月〜3月 |
| **目的** | 橋渡しシステムの第一歩 + AI連携の足がかり |
| **解決する課題** | データ分断、全体像が見えない |
| **やること** | Phase 1-3のデータを統合して可視化 |
| **成果物** | 統合ダッシュボード（部品在庫 + 製造委託先 + PSI + 入出庫履歴） |
| **オプション** | 欠品リスクアラート（AI/閾値ベース） |

**前提条件**:
- Phase 1-3が完了していること
- kintone API許可の状況（あれば活用、なければCSV継続）

**検討事項**:
- 将来のkintone統合に向けたデータ設計
- GEN巻き取りの方針が決まれば反映
