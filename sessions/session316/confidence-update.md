### 農業問合せのデータ構造（M4に直結）

| セクション | 内容 | 品質DB観点 |
|-----------|------|----------|
| お問い合わせ情報 | 基本情報、ステータス | 発生日・対象製品の特定 |
| 問合せ入力情報 | Webフォームからの自動反映 | 顧客からの申告内容 |
| 業推一次対応方針 | 一時対応方針 | 初期判断 |
| 解析結果（結論） | ログ解析結果 | 技術的原因 |
| **製品分類カテゴリ** | 事象が発生した製品カテゴリ | **分類集計に使用**（マニュアル明記） |
| **不具合内容分類** | 事象の内容分類 | **分類集計に使用** |
| **不具合原因分類** | 原因分類 | **分類集計に使用** |
| クローズ内容 | 最終報告内容 | 対策・結論 |
| 残課題 | 恒久対応の追跡 | 再発防止 |

→ **農業問合せアプリのCSV出力がM4の最重要データソースになる可能性が高い**

### SCM受入業務のデータ構造（M3に関連）

| ステップ | kintone上の操作 | 品質DB観点 |
|---------|---------------|----------|

## 推奨: 次のアクション

**CSV確認を先にやるべき理由**:
- 確認事項#1（AgriシートとkintoneのCSV突合）はCSV出力で自己完結できる
- 確認事項#10, #11, #12もCSVで確認できる
- ヒアリング前にデータで確認できることを済ませた方が、質問がピンポイントになる

**CSV出力の優先順位**:
1. **販売（農業）** — Agriシートとの突合。確度B→Aにできる可能性
2. **農業問合せ** — M4の不具合分類の選択肢確認
3. **受入製品** — M3の受入検査フィールド確認
4. **請求/請求明細** — PSI売上データとの関係確認
