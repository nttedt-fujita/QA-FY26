**作成**: Session 316 (2026-03-26)

---

## 1. kintone導入の背景

| 項目 | 内容 |
|------|------|
| **導入目的** | ①e-Drone内の全体売上を一元管理 ②チーム専用Excelをなくし部署間のやり取りを円滑化 |
| **導入経緯** | Salesforceで準備していたが予算とセキュリティの問題でkintoneに変更 |
| **環境** | NTT東日本が契約しているkintone環境内にe-Drone専用サイトを作成 |
| **サイト名** | 「【戦略子会社】NTT e-Drone Technology」 |
| **kintone管理チーム** | 和田駿、瀬川梓、宮崎雄大、猪又恵理 |

---

## 2. システム間の役割分担

| システム | 管理範囲 |
|----------|----------|
| **kintone** | 売上管理（受注後〜請求）、顧客管理、商談管理、発送管理、CS問合せ管理 |
| **GEN** | 在庫管理、資産管理、部材の発注管理（残る） |
| **Moneyforward** | 会計（入金確認・売上計上） |
| **formrun** | Webフォーム（問合せ受付、ログアップロード） |
| **SharePoint** | ファイル格納（ログファイル等、kintoneは容量制限のため） |

**重要**: kintoneからGENへは定期的にCSVインポートで販売在庫情報を連携している

## 3. 部署別の管理範囲

| 管理対象 | 内容 |
|----------|------|
| 取引先 | 法人情報 |

| 販売 | 販売管理 |

| スクール | スクール管理 |
| 請求 | 請求データ |
| 発送 | 発送管理 |

**業務フロー**: 引合い → 顧客登録 → 商談 → 見積書/注文書出力 → 受注報告 → 発送登録 → 請求データ取込

| 産業商談 | 機体販売・レンタル・受託案件 |
| 商談商品 | 提案商品・サービス（産業製品） |

### 3.3 SCM部門
**マニュアル**: KintoneSCM業務マニュアル（42スライド、2025年1月）

| 管理対象 | 内容 |
|----------|------|
| 発送 | 社有製品の発送情報 |
| 発送製品 | 発送する製品の詳細 |
| 受入製品 | 受入検査情報 |
| 機体 | 社有機体の状況管理 |

**発送データのフィールド**:
- 発送状況（発送完了等）
- 伝票番号
- 発送日
- 運送会社

**受入業務フロー**: 着荷 → シリアル検索 → 伝票確認 → 着荷情報登録 → 受入検査担当者通知 → 受入検査 → 梱包 → 完了

### 3.4 業務推進部門
**マニュアル**: Kintone農業問合せマニュアル（38スライド、2024年12月）

| 管理対象 | 内容 |
|----------|------|
| 農業問合せ | CS情報（Webフォーム問合せ〜対応完了） |
| 残課題 | 問合せから発生した残課題の管理 |
| 修理点検履歴 | 修理・点検情報 |

**農業問合せのセクション構成**:

| セクション | 登録内容 | 担当 |
|-----------|----------|------|
| お問い合わせ情報 | 基本情報（問合せ元・ステータス等） | 業推・開発 |
| 問合せ入力情報 | Webフォーム登録内容を反映 | ー |
| 業推一次対応方針 | 一時対応内容 | 業推 |
| 解析結果（結論） | ログ解析結果 | 開発・業推 |
| 原因 | 事象発生原因 | 開発 |
| 製品分類カテゴリ | 事象発生製品カテゴリ | 開発 |
| 不具合内容分類 | 事象内容分類 | 開発 |
| 不具合原因分類 | 原因分類 | 開発 |
| クローズ内容 | 最終報告内容（お客様向け） | 開発 |
| 関連レコード | 関連情報リンク | ー |

**問合せフロー**: フォーム受付 → 業推確認 → ヒアリング → ログ解析 → 代替品発送/修理 → E/U報告 → クローズ

### 3.5 経理部門
**マニュアル**: Kintone経理マニュアル（26スライド、2024年12月）

| 管理対象 | 内容 |
|----------|------|
| 請求 | 請求書ヘッダー（請求先/請求額合計/請求番号等） |
| 請求明細 | 請求書明細（製品/数量/単価/金額等） |
| 販売 | 販売データ |
| レンタル | レンタルデータ |
| 契約 | 契約データ |
| スクール | スクールデータ |

### 3.6 群馬通商（外部パートナー）
**マニュアル**: 群馬通商様_kintone操作マニュアル（52スライド、2025年3月28日）

| 操作範囲 | 内容 |
|----------|------|
| 発送完了登録 | 保管製品の発送完了を登録 |
| 作業情報登録 | バッテリ充放電作業の情報を登録 |

## 5. データフロー概要

```
[顧客]
  ↓ 問合せ/引合い
[kintone: 取引先/取引先担当者]
  ↓
[kintone: 商談（普及 or 産業）]
  ↓ 見積書/注文書出力
[kintone: 受注報告]
  ↓
  ├→ [kintone: 販売] ─→ [kintone: 発送] ─→ [群馬通商/SCM: 発送業務]
  ├→ [kintone: レンタル]
  └→ [kintone: スクール]
  ↓
[kintone: 請求/請求明細]
  ↓
[Moneyforward: 入金確認/売上計上]

[GEN: 在庫管理/資産管理] ←→ CSV連携 → [kintone: 販売在庫情報]
```

## 6. PSI分析に関連するデータの所在

Session 314の確度整理（B/C/D検証）の観点から、kintone内で関連するデータ:

| PSI要素 | kintoneでの対応アプリ | 備考 |
|---------|---------------------|------|
| **P（生産）** | 直接管理なし | GENの在庫管理側。群馬通商の入庫・製造S/N登録がヒントになる |
| **S（販売）** | 販売（農業/SoL）、産業商談 | 受注〜請求のデータ。商談商品に数量・単価あり |
| **I（在庫）** | 機体、オプション | SCM部門のアプリ。社有製品の状況管理。GENにも在庫あり |
| **発送** | 発送、発送製品 | 発送日・伝票番号・発送先。S（販売）の実績裏付け |
| **請求** | 請求、請求明細 | 請求先/請求額/請求番号。経理部門で管理 |
| **問合せ/不具合** | 農業問合せ | 不具合内容分類・原因分類あり。品質分析に有用 |
| **受入検査** | 受入製品 | SCM部門。受入検査情報あり |

## 7. スクリーンショット追加候補

kintone構造の把握に有用なスクリーンショット候補:

| 優先度 | 画面 | 取得理由 |
|--------|------|----------|
| **高** | 販売（農業）一覧 | PSI分析のS（販売）データ。フィールド構成を確認 |
| **高** | 販売（SoL/産業）一覧 | 同上。産業側の販売データ |
| **高** | 請求一覧 | 請求データのフィールド構成確認 |
| **高** | 農業問合せ一覧 | 不具合/品質データ。M4（工程不良DB）と関連 |
| **中** | 機体アプリ一覧 | 在庫（I）データのフィールド構成 |
| **中** | 商談（普及）一覧 | 商談のフィールド・ステータス構成 |
| **中** | 産業商談一覧 | 同上（産業側） |
| **低** | アプリ一覧（全件表示） | toppage3の「さらに表示」後。全アプリの把握 |
