## 2. アプローチの分類

| アプローチ | 説明 | 配布方法 | ユーザー操作 |
|-----------|------|---------|-------------|
| **Webサーバー型** | バックエンドがHTTPサーバー、ブラウザでアクセス | 実行ファイル起動 → ブラウザで `localhost:8080` 等を開く | ◎ 簡単 |
| **デスクトップアプリ型** | 専用アプリをインストール（Tauri, Electron等） | インストーラー配布 | ○ |

**判断**: R1「Webアプリ的なUI」「触る人がやりやすい」から **Webサーバー型** を採用。

## 5. UBXパース方式の比較

| 方式 | 信頼性 | 開発コスト | 対応言語 |
|------|--------|-----------|---------|
| **u-blox公式 ubxlib** | ◎ | 中（FFI必要な場合あり） | C |
| **pyubx2** | ○ | 低 | Python |
| **ublox-rs** | △ | 低 | Rust |
| **daedalean/ublox** | △ | 低 | Go |
| **自前実装** | ○（仕様書ベース） | 中 | 任意 |

### 5.1 u-blox公式 ubxlib

- **URL**: https://github.com/u-blox/ubxlib
- **言語**: C
- **提供元**: u-blox（メーカー公式）
- **対応**: ESP32, STM32, nRF5, Windows, Linux
- **信頼性**: ◎ メーカー公式のため最も信頼できる
- **課題**: 組み込み向け設計、PC向けWebアプリで使うにはFFI経由

### 5.2 pyubx2（Python）

- **URL**: https://github.com/semuconsulting/pyubx2
- **メンテナンス**: 活発（2026年も更新あり）
- **対応メッセージ**: 多数（NAV, MON, CFG等）
- **信頼性**: ○ 広く使われている
- **課題**: Python限定

### 5.3 ublox-rs（Rust）

- **URL**: https://github.com/ublox-rs/ublox
- **メンテナンス**: 中程度
- **対応**: Series 8向けに開発、他バージョン対応中
- **信頼性**: △ 個人OSS、F9P（Series 9）対応要確認
- **課題**: F9P固有メッセージの対応状況が不明

### 5.4 daedalean/ublox（Go）

- **URL**: https://pkg.go.dev/github.com/daedaleanai/ublox
- **対応**: u-blox 8 / u-blox M8
- **信頼性**: △ 個人OSS
- **課題**: F9P対応要確認

### 5.5 自前実装

- **信頼性**: ○ 仕様書（u-blox Interface Description）に従えば正確
- **開発コスト**: 中（必要なメッセージだけ実装）
- **メリット**: 言語に依存しない、必要な部分だけ実装
- **課題**: バグの可能性、テストで検証必要

**必要なUBXメッセージ（5-6個）**:

| メッセージ | Class | ID | 用途 |
|-----------|-------|-----|------|
| NAV-SIG | 0x01 | 0x43 | L1/L2別C/N0 |
| NAV-SAT | 0x01 | 0x35 | 衛星方位角・仰角 |
| NAV-PVT | 0x01 | 0x07 | 位置・RTK状態 |
| NAV-STATUS | 0x01 | 0x03 | TTFF |
| MON-SPAN | 0x0A | 0x31 | スペクトラム |
| MON-RF | 0x0A | 0x38 | ジャミング状態 |

## 6. 衛星位置計算について

### 6.1 当初の懸念

QZSS APIから軌道データを取得し、自前で衛星位置（方位角・仰角）を計算する方式を検討していた。

**問題点**:
- 衛星位置計算ライブラリ（skyfield, satellite.js等）の精度・信頼性が不明
- 計算ロジックのバグで誤った位置を表示するリスク

### 6.2 解決策

**F9PのUBX-NAV-SATメッセージから直接取得する**

- F9Pが計算した `azim`（方位角）、`elev`（仰角）をそのまま使用
- 衛星位置計算の信頼性問題を完全に回避
- パフォーマンス負荷も軽減

**QZSS APIの用途**:
- 予測（未来の衛星配置）が必要な場合のみ使用
- 通常の評価作業では不要
