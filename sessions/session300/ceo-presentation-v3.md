# Phase 1: 集計業務の自動化（4-6月）

## Before / After

| Before | After |
|--------|-------|
| kintone → **手動転記** → PSI | kintone → **自動取得** → PSI |
| 週2時間の作業 | 確認のみ |

## 前提条件（確認中）

- kintone API許可（NTT東基盤）または独自契約
- 宮崎さん・和田さんが対応中

---

# Phase 2: 入出庫の自動記録（7-9月）

## 現状の課題

| 課題 | 影響 |
|------|------|
| GEN/kintone両方に手動入力 | 二重管理 |
| どの棚に何があるか不明 | 探す時間が発生 |
| 入出庫履歴が追えない | FIFO管理不可 |

## 目指す姿

- 棚にQRコード貼付 → スキャンで自動記録
- どの棚に何があるか即座に分かる

## 導入コスト: **約1-3万円**（ラベル・プリンタ）

---

# Phase 3: 委託先在庫の見える化（7-12月）

## Before / After

| Before | After |
|--------|-------|
| 月末に「しそん情報」が届く | 週次/日次で報告 |
| 月中は見えない | リアルタイム可視化 |

## 前提条件

- 製造委託先との連携方法の合意
- 報告頻度の決定（日次/週次）

---

# Phase 4: SCM全体の一元管理（1-3月）

## 統合ダッシュボードで一覧表示

| データ | 内容 |
|--------|------|
| 部品在庫 | 入庫/出庫/残数/ロット |
| 製造委託先 | 実在庫・組立進捗 |
| 製品在庫 | 組立機体/出荷準備 |
| PSI | 需要・供給・在庫 |

## オプション：欠品リスクアラート

- 在庫が閾値を下回ったら自動通知
