# Session 166 サマリー

**日付**: 2026-03-13

**目的**: 待機時間の最適化検討（因果関係の検証）

---

## 実施内容

### 1. 因果関係の整理

Session 164-165で「200ms→500msに変更 + 115200bps化」の2つを同時に変えていた。
正しく検証するため、38400bpsに戻して待機時間のみを変数として検証。

### 2. テスト実施

| 条件 | 結果 |
|------|------|
| 38400bps + 500ms | 5/5 成功 |
| 38400bps + 200ms | 5/5 成功 |
| 38400bps + 100ms | 3/5 成功（2回失敗）|

### 3. 失敗パターン

100msでは送信バッファ排出が間に合わず、以下の連鎖的失敗:
- NMEA OFF送信タイムアウト
- NAV-PVT, NAV-STATUS, NAV-SAT, NAV-SIG送信タイムアウト
- 1回おきに失敗（NTRIP転送との競合）

---

## 結論

- **38400bpsの最小待機時間**: 200ms
- **境界**: 100〜200msの間
- **現状維持**: 500ms（安全マージン込み）
