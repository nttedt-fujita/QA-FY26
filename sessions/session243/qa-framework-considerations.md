## 1. IQCでの位置づけ

AI検査システムは**IQC（受入検査）の効率化**に位置する。

```
サプライヤ → [IQC: 受入検査] → 製造工程 → [PQC] → [OQC] → 顧客
                 ↑
              AI検査はここ
```

---

## 2. AQL（抜取検査）との関係

**ステータス**: 要相談

| 観点 | 考慮事項 |
|------|----------|
| 全数 vs 抜取 | AI検査は「全数検査の代替」か「抜取検査の判定支援」か |
| 部品依存 | 部品・背景文脈によってAQL基準が切り替わる可能性 |
| 撮影コスト | 抜取なら撮影枚数も減る |

→ **ヒアリングで確認**

---

## 3. トレーサビリティ

**ステータス**: 必要

### ロット概念について（過去セッションの決定事項）

- **現状**: ロットという概念が存在しない
- **方針**: 入荷タイミングをロットとして定義する
- **M3/M4ドメインモデリング**: 同じ視点で設計済み

### AI検査での活用

- AI検査結果をロット単位で記録
- 「このロットでシルク印刷カケが多発」→ サプライヤへフィードバック
- 不良発生時のトレースバックに使用可能

---

## 4. 8Dフレームワークとの関係

| 8Dステップ | AI検査での活用 |

| D2: 問題の明確化 | AI検出した不良の記録 |
