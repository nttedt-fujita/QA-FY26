## 1. PSI管理とは

### 定義

**PSI管理** = Production（生産）・Sales（販売）・Inventory（在庫）の3つの計画を同時・一体で管理する手法。

```
需要予測 / 受注情報
        ↓
[S: 販売計画] → どれだけ、いつ売れる見込みか
        ↓
[I: 在庫計画] → 販売計画と生産計画の差分を在庫で吸収
        ↓
[P: 生産計画] → どれだけ、いつ作るか（調達・製造）
```

> **ポイント**: 3つを別々に最適化しない。販売だけ優先 → 欠品、生産だけ優先 → 過剰在庫が発生する。

### 目的

| 課題 | PSI管理での対処 |
|------|----------------|
| 欠品（販売機会損失） | 販売計画に基づき在庫・生産を先読み |
| 過剰在庫（保管コスト・廃棄リスク） | 需要予測に連動した適正在庫の維持 |
| 部門間の計画不整合 | 販売・製造・調達を同一テーブルで調整 |

---

## 2. PSI管理の実施サイクル

### 標準的な運用フロー

```
1. 販売計画の策定（営業）
   ↓
2. 在庫・仕入れ計画の策定（調達・倉庫）
   ↓
3. 生産計画の策定（製造）
   ↓
4. 計画と実績の差異確認
   ↓
5. アクション決定（増産・調達先変更・値引き等）
   ↓ （繰り返し）
```

### 管理頻度

| 頻度 | 向いている場面 |
|------|----------------|
| **月次** | 中長期安定運用、経理実績との連動 |
| **週次** | 需要変動が大きい商材、高頻度調整が必要な場合 |

> **S&OP（Sales & Operations Planning）**: PSI管理のグローバル版。財務計画とも統合し、キャッシュフロー・設備投資最適化まで含む。

### 差異管理

PSIの実績と計画の差異を毎サイクルで確認し、対応方針を決定する。

| 差異パターン | 考えられる原因 | アクション例 |
|-------------|---------------|-------------|
| 実績販売 < 計画 | 需要予測の過大、競合影響 | 生産縮小、価格調整 |
| 実績販売 > 計画 | 需要予測の過小 | 緊急増産、代替調達 |
| 在庫 > 目標 | 生産過多、販売不振 | 販促強化、生産調整 |
| 在庫 < 目標 | 欠品リスク | 緊急発注、生産優先 |

## 3. PSIの各計画の役割と指標

### P（生産計画）

**役割**: 在庫計画に基づいて、いつ・どれだけ製造するかを決める。

| KPI | 説明 |
|-----|------|
| 生産量達成率 | 計画生産量に対する実績生産量の割合 |
| 製造リードタイム | 生産着手から完成までの所要日数 |
| 設備稼働率 | 実稼働時間 ÷ 計画稼働時間 |

### S（販売計画）

**役割**: 需要予測・受注情報をもとに、いつ・どれだけ売れるかを見積もる。

| KPI | 説明 |
|-----|------|
| 需要予測精度 | (1 − abs(予測誤差) ÷ 実績) × 100% |
| 受注充足率 | 納期通りに納品できた受注の割合 |
| 欠品率 | 在庫切れにより出荷できなかった件数の割合 |

### I（在庫計画）

**役割**: 販売計画と生産計画のバッファ。適正在庫を維持する。

| KPI | 説明 | 目安 |
|-----|------|------|
| 在庫回転率 | 在庫の回転速度 | 製造業: 10〜13回/年 |
| 在庫日数 | 現在の在庫が何日分か | 在庫回転率から算出 |
| 過剰在庫率 | 適正在庫を超えた在庫の割合 | 低いほどよい |
| 安全在庫 | 欠品を防ぐための最低保有量 | リードタイム・需要変動で決定 |

## 4. 在庫管理指標の計算式

### 4-1. 在庫回転率

**意味**: 一定期間内に在庫が何回入れ替わったか（高いほど効率的）

```
在庫回転率 = 売上原価（出庫金額） ÷ 平均在庫金額
平均在庫金額 = (期首在庫 + 期末在庫) ÷ 2
```

**業界参考値（製造業）**:

| 区分 | 回転率 | 在庫日数換算 |
|------|--------|-------------|
| 製造業 全体平均 | 約11回/年 | 約33日 |
| 中小製造業平均 | 約12.6回/年 | 約29日 |
| 大企業製造業平均 | 約10.5回/年 | 約35日 |

> **出典**: 経済産業省 商工業実態基本調査

**在庫回転日数** = 365 ÷ 在庫回転率

例: 回転率12回 → 在庫日数30日（月1回で全在庫が入れ替わる）

---

### 4-2. 安全在庫

**意味**: 需要変動やリードタイムのばらつきによる欠品を防ぐための最低保有量

```
安全在庫 = 安全係数 × 使用量の標準偏差 × √(発注リードタイム + 発注間隔)
```

**安全係数（サービス率と欠品許容率の対応）**:

| 欠品許容率 | サービス率 | 安全係数 |
|-----------|-----------|---------|
| 10% | 90% | 1.28 |
| 5% | 95% | **1.65** （標準） |
| 2% | 98% | 2.05 |
| 1% | 99% | 2.33 |
| 0.1% | 99.9% | 3.09 |

**計算例**:
- 条件: 欠品許容率5%（安全係数1.65）、使用量標準偏差20個、リードタイム5日、発注間隔6日
- 安全在庫 = 1.65 × 20 × √(5+6) = 1.65 × 20 × 3.32 ≈ **110個**

> 標準偏差はExcelの`STDEV関数`で過去実績から算出可能。

---

### 4-3. 適正在庫

```
適正在庫 = 安全在庫 + サイクル在庫
サイクル在庫 = 1発注期間中の平均使用量 ÷ 2
```

## 5. 自社への適用（DX効果分析の観点）

### 現状の課題（調査済み）

| 課題 | 確認セッション | 内容 |
|------|--------------|------|
| 在庫タイムラグ最大1週間 | S321 | GENへの反映が週次手動インポートのため遅延 |
| kintone→GEN手動インポート | S322 | 週次手作業、工数発生 |
| 群馬通商Excel手作業 | S327 | kintone情報を見て手入力でフラグ・レコード番号管理 |

### PSI管理の観点での課題整理

```
現状フロー:
kintone（出荷実績） → 週次手動インポート → GEN（在庫・PSI）
                          ↑ここが遅延源

PSI管理の「I（在庫計画）」がリアルタイムでない
→ 在庫情報が最大1週間古い状態でS・Pの計画を立てている
```

**DX効果の仮説**（定量化はQA-24で）:
- 自動化によるタイムラグ解消 → 在庫計画精度向上 → 欠品・過剰在庫の削減
