# Hooks導入ニーズ分析 - 質問リスト

## 目的

`~/.claude`ディレクトリにhooksを導入し、作業効率・トークン効率を向上させる。
効果の高いhooksを特定するため、現状の課題とニーズを洗い出す。

---

## 質問1: 繰り返し作業の特定

**質問**: 普段の作業で「これClaudeにやらせなくていいのに」と感じる繰り返し作業はありますか？

**例**:
- フォーマット（prettier）
- lint（ESLint、ruff）
- ファイル存在確認
- git status確認

**なぜ知りたいか**:
PostToolUseフックで自動化できる候補を特定するため。
機械的な繰り返し作業をhooksに追い出せば、その分のトークン消費がゼロになる。

**関連するhooks**: PostToolUse

---

## 質問2: トークン無駄遣いの場面

**質問**: トークンを無駄遣いしているな、と感じる場面はありますか？

**例**:
- Claudeが毎回セッション履歴を調べに行く
- 同じファイルを何度も読みに行く
- 毎回同じ情報を確認させられる

**なぜ知りたいか**:
UserPromptSubmitフックでコンテキスト自動注入すれば、
Claudeが「調べに行く→読む→判断する」の往復をスキップできる。
A部分（調べるコスト）の削減が可能。

**関連するhooks**: UserPromptSubmit, SessionStart

---

## 質問3: 失敗→やり直しパターン

**質問**: 「Claudeが勝手にやって失敗して、やり直し」になるパターンはありますか？

**例**:
- テストなしでコード書いて、後からテスト落ちる
- 触っちゃいけないファイルを触る
- 本番設定を変更してしまう
- 間違った方向に進んでから引き返す

**なぜ知りたいか**:
PreToolUseフックで早期ブロックすれば、手戻りコストを削減できる。
間違った方向に進む前に止めることで、数ターン分のトークン浪費を防げる。

**関連するhooks**: PreToolUse

---

## 質問4: 品質確認の手間

**質問**: 作業完了後に「テスト走らせた？」「ビルド通る？」と確認する手間はありますか？

**例**:
- Claudeが「完了しました」と言った後に、人間がテスト実行を依頼
- ビルド確認を毎回指示する必要がある

**なぜ知りたいか**:
Stopフックで自動テスト実行すれば、人間の確認ターンが不要になる。
失敗したらClaudeが自動で修正を継続する。

**関連するhooks**: Stop

---

## 質問5: 現在のSessionStartフックの評価

**質問**: 現在のSessionStartフック（energy/session-managementを促す）は機能していますか？改善点はありますか？

**現在の動作**:
```
[SessionStart Hook]
セッション開始を検知しました。

以下を実施してください:
1. energy-management スキルで体調チェック
2. session-management スキルで前回コンテキスト確認

参照: ~/.claude/CLAUDE.md
```

**なぜ知りたいか**:
既存フックの効果を評価し、改善の余地を探る。
「促す」だけでなく「自動で情報を注入する」形に進化させられるか検討。

**関連するhooks**: SessionStart

## 質問6: プロジェクト固有 vs グローバル

**質問**: hooks導入は `~/.claude`（グローバル）のみですか？それともプロジェクト単位でも設定しますか？

**選択肢**:
- A: グローバルのみ（全プロジェクト共通）
- B: グローバル + プロジェクト固有（プロジェクトごとに追加設定）

**なぜ知りたいか**:
設定ファイルの配置場所と、スクリプトの汎用性を決めるため。
グローバルなら `~/.claude/hooks/` に汎用スクリプト、
プロジェクト固有なら `.claude/hooks/` にプロジェクト専用スクリプト。

**関連するファイル**: settings.json, settings.local.json

## 質問7: 使用言語・ツール

**質問**: 普段の開発で使用している言語・ツールは？

**例**:
- Python + ruff/black
- TypeScript + prettier/ESLint
- Go + gofmt
- Rust + rustfmt

**なぜ知りたいか**:
PostToolUseでフォーマット自動実行する場合、対象言語・ツールを特定する必要がある。

**関連するhooks**: PostToolUse
