## 2. 優先順位の推奨案

### 推奨: M3/M4先行 → M1/M2並行

```
FY2026
  1Q        2Q        3Q        4Q       FY2027 1Q
  ├─────────┼─────────┼─────────┼─────────┼─────────┤
  │← M3/M4 Step1 →│← M3/M4 Step2 →│← M3/M4 Step3 →│
  │ 要件定義・設計   │ DB構築・移行    │ 運用開始       │
  │                │                │                │
  │← M1 Step1 ──→│← M1 Step2 ──────────→│← M1 Step3→│
  │ 評価手法調査     │ 評価環境構築・実施      │ 標準化    │
  │                │                │                │
  │                │← M2 Step1 →│← M2 Step2/3 ──→│
  │                │ 指標定義     │ パイプライン構築  │
```

### 推奨理由

**M3/M4を先に着手すべき理由**:

1. **ソフトスキルが最も直接的に活きる**
   - DB設計、SQL、データ分析は藤田さんの既存スキルそのもの
   - 技術的な不確実性が低く、確実に前進できる

2. **早期に成果が出せる**
   - 「新人が来て、すぐ目に見える仕組みを作った」という実績
   - 品質G内での信頼構築につながる

3. **他メンバーの業務を支援できる**
   - いしかわのEVPEAK等継続評価データをDB化
   - こいたばしのH12G評価データもDB化
   - 品質G全体のデータドリブンな意思決定を支える

4. **M3/M4は一体設計すべき**（Session 1で発見済み）
   - lot_idで紐付けることでサプライヤ品質と工程不良の相関分析が可能
   - 同時に設計することで整合性が取れる

5. **M1はスケジュール上も1Q開始だが、並行で調査フェーズは進められる**
   - Step1（評価手法調査・計画策定）は座学・調査が中心
   - M3/M4のDB設計と並行して、M1の情報収集・規格学習は可能

### 注意点

- スライド2のM1スケジュールは1Q開始。**完全に後回しにはできない**
- M1 Step1（調査・計画）は軽い作業量でM3/M4と並行可能
- M2はM1の成果物が入力になる部分があるため、少し遅れてスタートでOK
