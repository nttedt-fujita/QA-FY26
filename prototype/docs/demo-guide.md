# デモガイド

M3 受入検査DB プロトタイプのデモ手順。

## デモの目的

現場での受入検査業務をデジタル化するプロトタイプを体験してもらい、フィードバックを得る。

## デモ準備

```bash
cd prototype
make demo-setup
# 別ターミナルで
make frontend-dev
```

## デモフロー

### 画面1: ロット登録 (http://localhost:3000)

**操作手順**:
1. 部品ドロップダウンから部品を選択
   - 例: 「ブラシレスモーター M2006」
2. サプライヤー名が自動表示されることを確認
3. ロットIDを入力（例: `LOT-2026-001`）
4. 数量を入力（例: `100`）
5. 「登録して検査開始」ボタンをクリック

**ポイント**:
- 部品選択でサプライヤーが自動表示 → マスタ連携の確認

---

### 画面2: 検査入力 (/inspection)

**操作手順**:
1. 作業者ドロップダウンから作業者を選択
2. 検査項目ごとにOK/NG/SKIPボタンで入力
   - 「OK」を連打してみる
   - 「NG」を押してみる
   - 「取り消し」で直前の入力を取り消し
3. カウンターの数値が変わることを確認
4. 「検査完了」ボタンで終了

**ポイント**:
- **カウンター方式**: 「合格、合格、合格...」という現場作業に合わせた設計
- **取り消し機能**: 打ち間違いを簡単に修正
- **工数自動計算**: 開始〜終了の時間を自動記録

---

### 画面3: 検査一覧 (/records)

**操作手順**:
1. 登録した検査記録が一覧に表示されることを確認
2. フィルターで絞り込み
   - 部品で絞り込み
   - 結果で絞り込み（合格/不合格）
3. CSVエクスポートボタンでダウンロード

**ポイント**:
- 検索・フィルター機能
- CSVエクスポート → Excelで分析可能

---

### 画面4: ダッシュボード (/dashboard)

**操作手順**:
1. KPI欄を確認
   - 総検査数
   - 不良率
   - 今月の検査数
2. 月別グラフを確認
3. 検査項目別の不良率を確認
4. サプライヤー別の不良率を確認

**ポイント**:
- リアルタイムで集計表示
- サプライヤー比較 → 品質問題の傾向把握

---
