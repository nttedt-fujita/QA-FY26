# GNSS評価ツール ドメインモデル

**作成日**: 2026-03-10
**Session**: 62

---

## 1. 目的

受信感度、衛星位置、RTK FIX時間、精度、ジャミング状態を**自動記録**し、手入力・転記ミスを排除する。

---

## 2. ドメイン概念

### 2-1. エンティティ一覧

| エンティティ | 説明 | 主な属性 |
|-------------|------|---------|
| デバイス | 評価対象のGNSSモジュール | 型番、シリアル番号、メモ |
| 計測セッション | 1回の評価作業 | 開始/終了時刻、評価者、評価条件、評価結果 |
| 位置・状態 | NAV-PVTから取得 | lat, lon, hAcc, vAcc, carrSoln, numSV |
| FIX状態 | NAV-STATUSから取得 | ttff, fixOk |
| DOP | NAV-DOPから取得 | pDOP, hDOP, vDOP |
| 衛星 | NAV-SATから取得 | svId, gnssId, elev, azim |
| 信号 | NAV-SIGから取得 | sigId, cno, quality |
| スペクトラム | MON-SPANから取得 | spectrum[256], center, span |
| RF状態 | MON-RFから取得 | jammingState, antStatus |

### 2-2. 関係図

```
デバイス（評価対象）
    │
    └── 計測セッション (1:N)
            │
            ├── 評価条件
            │     - 場所（緯度、経度、説明）
            │     - 環境（屋外/屋内、天候）
            │     - アンテナ構成
            │     - 基地局情報（RTK評価時）
            │
            ├── 計測データ（時系列）
            │     ├── 位置・状態 (NAV-PVT) (1:N)
            │     ├── FIX状態 (NAV-STATUS) (1:N)
            │     ├── DOP (NAV-DOP) (1:N)
            │     ├── 衛星 (NAV-SAT) (1:N)
            │     │     └── 信号 (NAV-SIG) (1:N per 衛星)
            │     ├── スペクトラム (MON-SPAN) (1:N)
            │     └── RF状態 (MON-RF) (1:N)
            │
            └── 評価結果
                  - 判定（合格/不合格/保留）
                  - 所見（コメント）
```

## 3. スコープ判断

### 3-1. プロトタイプ段階でのスコープ

| 項目 | スコープ | 理由 |
|------|---------|------|
| 集計・統計値 | **都度計算** | データ量が少ない、保存は将来検討 |
| 評価基準 | **設定ファイル** | 変更しやすい、DBマスタは過剰 |
| レポート出力 | **スコープ外** | 画面表示のみ、将来検討 |

### 3-2. 将来の拡張候補

- 集計値の永続化（大量データ対応）
- 評価基準のDB管理（履歴管理、複数基準対応）
- レポート出力（PDF、Excel）

## 4. 検証方法

実際の評価作業フローを追って概念を洗い出した。

| ステップ | 必要な概念 | 対応 |
|---------|-----------|------|
| 1. デバイス用意 | デバイス（型番、S/N） | ✓ |
| 2. 環境セットアップ | 評価条件、アンテナ、基地局 | ✓ |
| 3. 計測開始 | セッション開始時刻 | ✓ |
| 4. データ収集 | 各UBXデータ | ✓ |
| 5. 計測終了 | セッション終了時刻 | ✓ |
| 6. データ分析 | 集計・統計値 | 都度計算 |
| 7. 判定 | 評価基準、判定結果 | 設定ファイル + 評価結果 |
| 8. レポート | レポート出力 | スコープ外 |
