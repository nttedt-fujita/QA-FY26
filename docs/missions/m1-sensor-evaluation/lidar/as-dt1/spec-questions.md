### 1.7 出力データフォーマット（旧Q03 解決）

| 形式 | データ長/点 | 含まれるデータ | 備考 |
|------|------------|--------------|------|
| ascii | 10 bytes | Z方向のみ、1次ピーク | `%08.2f CR LF` フォーマット |
| binary | 15 bytes (120bit) | XYZ点群、1次・2次ピーク | 固定小数点20bit、0.25mm分解能 |
| binz | 2.5 bytes (20bit) | Z方向のみ、1次ピーク | 固定小数点20bit、0.25mm分解能 |

**必要データレート（30MSTD, fsync 33.33ms時）**:
- ascii: 1.4 Mbps
- binary: 2.1 Mbps
- binz: 0.35 Mbps

**座標系**: 垂直方向下向き=X+、水平方向左=Y+、測距方向=Z+

出典: APIマニュアル p6, p9-10

### 1.10 障害物検出機能

| 項目 | 値 |
|------|-----|
| 検出範囲設定 | 0〜50000 mm |
| 検出モード | 4パターン（範囲内、近距離、遠距離、範囲外） |
| 出力 | Trigger Out端子 または obsコマンド |

出典: APIマニュアル p30

### 1.11 IMU（加速度・角速度センサー）

| 項目 | 仕様 |
|------|------|
| サンプリング | 約100Hz（10ms間隔） |
| 加速度レンジ | ±4G、18bit |
| 角速度レンジ | ±500 dps、16bit |
| ノイズ（GYRO） | 3.8 mdps/√Hz |
| ノイズ（ACCEL） | 70 µG/√Hz |

**注意**: 非加工データ。ユーザー側でキャリブレーション必要。

出典: APIマニュアル p31-32

### 1.12 時間同期

| クロック | 分解能 | ビット幅 | 用途 |
|---------|--------|---------|------|
| 測距用クロック | 0.5µs | 32bit | HISTモードEBDデータ |
| CPUクロック | 1ms | 32bit | 測距データ、IMUデータ |
| IMUクロック | 1µs | 16bit | IMUデータ内 |
| タイムスタンプ | 最小1ms | 32bit | TS Clk入力で歩進 |

出典: APIマニュアル p33

### 1.13 トリガー

| 種類 | 説明 |
|------|------|
| ソフトウェアトリガー | `t`コマンドで1回測距 |
| セルフトリガー | 内部クロックで繰り返し測距（fsync設定） |
| ハードウェアトリガー | Trigger In端子のエッジで測距開始 |

**遅延**: トリガー入力→測距開始: 25±10µs

出典: APIマニュアル p7-8

### 1.14 デバイス検索API

| 関数 | 機能 |
|------|------|
| get_uvc_list() | UVCデバイス番号とシリアル番号を取得 |
| get_uart_list() | UARTポートとシリアル番号を取得 |
| get_port_from_serial_number() | シリアル番号からポート取得 |

出典: APIマニュアル p77-78

## 5. 補足・備考

### 5.1 既存プロジェクトでの使用実績

当社では `SONY-LiDAR-TestMinutes` プロジェクトでAS-DT1を使用した実績あり:

- **ROS2ドライバー**: vendor提供のドライバーで動作確認済み（`as_dt1_ros2_driver`）
- **USBインターフェース**: `/dev/ttyACM0`, `/dev/ttyACM1` として認識
- **点群出力**: x, y, z（FLOAT32）、576点/フレーム
- **座標系**: X+=下、Y+=左、Z+=前（射出方向）— APIマニュアルと整合
- **30M15Fモード**: 屋外フィールドテストで使用、15fps動作確認
