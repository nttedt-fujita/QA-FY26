7
CDC モード／EXTUART モード
CDC モード、およびEXTUART モードは、本機の測距機能を使用するためのモードです。
CDC モードでは、一般的なOS に含まれているUSB CDC/ACM ドライバーで接続することができます。
EXTUART モードでは、8 ピンコネクターのUART 端子を使用して、一般的な非同期シリアル通信で接続することができます。
測距トリガー
測距を開始するトリガーについて説明します。
本機は、下記の方法で測距を開始させることができます。
ソフトウェアトリガー
USB CDC やUART からt コマンドにより1 回の測距を行います。
測距間隔の時間精度はホスト側のプログラムが、本機にt コマンドを送り出すタイミング精度に依存します。
測距および測距結果出力中にt コマンドを送った場合、トリガーは行われず無視されます。
セルフトリガー
本機の内部クロックを基準として、設定した間隔で測距を繰り返します。
測距間隔の設定は、fsync コマンドまたはflfsync コマンドを使用して行います。
測距レンジで決まる最小フレーム間隔未満の値（66.66 または33.33）を設定した場合は、最短のフレーム間隔で繰り返し測距が行わ
れます。
設定したフレーム間隔よりも測距間隔が伸びることがあります。これは、USB モードでホスト機器の処理速度が遅く、データの処理
が間に合わない場合、フレーム間隔を伸ばすことで遅延やハングアップを防ぐための処置です。
ハードウェアトリガー
8 ピンコネクターのTrigger In（3 ピン）に加えたパルスのエッジで測距を開始します。
◆Trig In 端子の信号レベルについては、ユーザーズガイドをご覧ください。
Trigger In 端子の設定は、trgin やfltrgin コマンドで行います。工場出荷状態では、disable です。
測距中にパルスを加えた場合、トリガーは行われず無視されます。
トリガー信号のエッジ入力から実際に測距が始まるまで、25±10 µs かかります。

トリガー出力
測距開始タイミングを出力するトリガー出力端子について説明します。
本機の8 ピンコネクターのTrigger Out 端子から測距に同期したパルスを出力することができます。

9
測距結果出力形式
測距結果の出力形式は、flformat やformat で設定できます。
設定することができる形式は、ascii、binary、およびbinz の3 種類です。
測距結果の改行コードはCR(0x0d)LF(0x0a) です。
測距結果出力は、下記のヘッダーからスタートします。
tm：本機のCPU クロックで計測された時刻情報 [ms]
ts：タイムスタンプカウンターの値
temp：測距時点のセンサー温度 [ ℃]
tm およびts は、測距開始時点の値を出力します。
測距結果出力の最後には、下記のフッターが付与されます。
USB SS モードで接続し、ホスト機器の通信性能が十分高速である場合は、binary モードを使用することで、点群を直接本機から出
力することができます。USB HS やUART で接続する場合や、ホスト機器の通信性能が低速な場合は、ascii モードやbinz モードで
出力することで、必要な通信速度を下げることができます。各出力形式で必要なデータレートは下記のとおりです。
測距レンジが30MSTDモード、fsyncが33.33の場合の必要データレート（概算値）
ascii形式
測距結果のZ 方向のみを整数部5 桁、小数点以下2 桁で、リーディングゼロサプレスしない形式で出力します。BEGIN MP から
END までの間は、5760 バイト、または2880 バイトの固定長です。測距できなかった点は、00000.00 の値が出力されます。
binary形式
測距結果を点群（X、Y、Z 座標）として出力します。1 点当たりのデータ長は15 バイト120 ビットで、BEGIN MP からEND までの
間は、8640 バイト、または4320 バイトの固定長です。X、Y、Z の各データはビッグエンディアン、固定小数点符号付き、20 ビッ
トで出力されます。値はmm 単位、小数点の位置は、ビット1 と2 の間で、0.25 mm 分解能で出力されます。1 測距点は、2 つの座
標値（1st、2nd）で構成されています。1st は最も多く測距のための光が戻ってきた点、2nd は次に多く光が戻ってきた点です。有
効な測距データがない点は、X、Y では0、Z では0x00000（0.00 mm）が出力されます。
1 点のデータのフォーマットは以下のとおりです。

複数台の本機を組み合わせて画角を拡大することができます。
本機の出力する本機を基準とした点群を、取り付けたシステムの基準点や基準となる方向に合わせて移動、回転させることで点群を
統合することができます。
本機のスキャンは、本機の基準面に平行に、順に測距が行われます。複数の本機の出すレーザーが影響を与えないように、本機を90
度回転させて、測距が垂直方向に同時に行われるように取り付けることを推奨します。
また、本機の垂直方向の測距可能範囲は約30 度なので、本機を30 度ずつ角度を変えて取り付けることで、死角の少ない測距を行う
ことができます。
本機の測距点は、30/23=1.3 度間隔で並ぶため、取付角度誤差が±0.5 度以下とすることを推奨します。
設定例：
3 台の本機を右に90 度傾けて30 度おきに水平に並べたとき、それぞれの点群出力に対して、次の処理を行うことで統合可能な点群
を生成することができます。
Rotation_xyz は下記のような関数です。

import numpy as np 
def Rotation_xyz(pcd, theta_x=0, theta_y=0, theta_z=0):
  theta_x = math.radians(theta_x)
  theta_y = math.radians(theta_y)
  theta_z = math.radians(theta_z)
  rot_x = np.array([[ 1,                 0,                  0],
                      [ 0, math.cos(theta_x), -math.sin(theta_x)],
                      [ 0, math.sin(theta_x),  math.cos(theta_x)]])
  rot_y = np.array([[ math.cos(theta_y), 0,  math.sin(theta_y)],
                      [                 0, 1,                  0],
                      [-math.sin(theta_y), 0, math.cos(theta_y)]])
  rot_z = np.array([[ math.cos(theta_z), -math.sin(theta_z), 0],
                      [ math.sin(theta_z),  math.cos(theta_z), 0],
                      [                 0,                  0, 1]])
  rot_matrix = rot_z.dot(rot_y.dot(rot_x))
  r = pcd @ rot_matrix
  return r
