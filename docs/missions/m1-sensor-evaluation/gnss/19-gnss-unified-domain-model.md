## 2. 統合ドメインモデル

```
ロット（入荷単位）
  │
  ├── 期待値（屋内検査用）
  │     - 出力レート（例: 100ms）
  │     - ポート設定（例: UBX+NMEA）
  │
  └── 装置 (1:N) ← シリアル番号で識別
        │
        ├── 屋内検査 (1:N) ← 同じ装置を複数回検査できる
        │     │
        │     ├── 検査日時
        │     │
        │     └── 検査項目結果 (1:N)
        │           - 通信疎通 → Pass/Error
        │           - シリアル番号 → Pass/Error
        │           - FWバージョン → Recorded（後で比較）
        │           - 出力レート → Pass/Fail
        │           - ポート設定 → Pass/Fail
        │
        └── 屋外計測 (1:N) ← 屋内検査に合格した装置のみ
              │
              ├── 計測条件
              │     - 場所（緯度、経度、説明）
              │     - 環境（天候）
              │     - 基地局情報（RTK評価時）
              │
              ├── 計測データ（時系列）
              │     ├── NAV-PVT（位置・精度）
              │     ├── NAV-STATUS（TTFF、FIX状態）
              │     ├── NAV-DOP（精度劣化）
              │     ├── NAV-SAT（衛星情報）
              │     └── MON-RF（ジャミング）
              │
              └── 評価結果（判定、所見）
```

---

## 3. エンティティ一覧

| エンティティ | 説明 | 主な属性 |
|-------------|------|---------|
| **ロット** | 同一仕様の装置群（入荷単位） | ロット番号、期待値（Rate、Port） |
| **装置** | 個別のGNSSモジュール | シリアル番号（SEC-UNIQID）、所属ロット |
| **屋内検査** | 1回の屋内検査作業 | 検査日時 |
| **検査項目結果** | 各項目の結果 | 項目、実測値、期待値、判定 |
| **屋外計測** | 1回の屋外計測作業 | 計測条件、開始/終了時刻 |
| **計測データ** | 時系列のUBXデータ | 各UBXメッセージのパース結果 |

---

## 4. 運用フロー

```
1. ロット登録（期待値を設定）
     ↓
2. 装置をUSB接続
     ↓
3. 屋内検査
   - ロットを選択
   - 各項目を実行
   - 結果を保存
   - → 合格なら次へ / 不合格なら別途対応
     ↓
4. 屋外計測（合格装置のみ）
   - 計測条件を設定
   - データ収集開始
   - データ収集終了
   - 結果を保存
     ↓
5. 結果確認・レポート
```

---

## 5. 検査項目と計測項目

### 屋内検査（設定確認）

| 項目 | UBXメッセージ | 判定方式 |
|------|-------------|----------|
| 通信疎通 | 応答確認 | Pass/Error |
| シリアル番号 | SEC-UNIQID | Pass/Error |
| FWバージョン | MON-VER | Recorded（後で比較） |
| 出力レート | CFG-RATE | Pass/Fail |
| ポート設定 | CFG-PRT | Pass/Fail |

### 屋外計測（性能評価）

| 項目 | UBXメッセージ | 計測方式 |
|------|-------------|----------|
| RTK FIX時間 | NAV-STATUS | 時系列記録 + 基準値比較 |
| 位置精度 | NAV-PVT | 時系列記録 + 統計 |
| 精度劣化 | NAV-DOP | 時系列記録 |
| 衛星捕捉 | NAV-SAT | 時系列記録 |
| ジャミング | MON-RF | 時系列記録 |

---

## 6. 共通項目（屋内・屋外）

| 項目 | 屋内 | 屋外 | 備考 |
|------|------|------|------|
| シリアル番号 | ✅ 取得 | ✅ 確認 | 紐づけのキー |
| FWバージョン | ✅ 取得 | - | 屋内で確認済み |
| 通信疎通 | ✅ 確認 | ✅ 確認 | 屋外でも再確認 |

---

## 7. M3との整合性（将来の統合用メモ）

| M3の概念 | M1での対応 | 備考 |
|---------|-----------|------|
| parts（部品） | GNSS | M3の1部品として扱える |
| lots（ロット） | ロット | 同じ概念 |
| inspection_records | 屋内検査 + 屋外計測 | 検査タイプで区別 |
| defect_reports | - | 将来追加可能 |

**統合時のポイント**:
- M1の「装置」はM3にはない（GNSS固有）
- M1の「屋外計測」もM3にはない（GNSS固有）
- 共通なのは「ロット」と「検査記録」の構造
