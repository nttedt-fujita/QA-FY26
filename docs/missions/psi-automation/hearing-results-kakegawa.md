## 2. 出荷関連

### 2.1 kintone → PSI転記

- **発送日ベース**で記入
- kintoneの**発送依頼通知**の情報から転記:
  - 発送先名
  - 発送日
  - レコード番号 → **ここだけ群馬通商の住所の所に書いている**（特殊な配置）

### 2.2 群馬通商Excelへの記載

- kintoneから群馬通商Excelにも転記:
  - 発送先名 / 発送日 / レコード番号
  - レコード番号 → **ここも群馬通商の住所の所に書いている**

### 2.3 群馬通商Excel → PSI転記

- **週の最後に**群馬通商Excelから転記
- 群馬通商Excelの**日程絞り込み機能**を使用して期間を絞っている

### 2.4 kintone CSV（09-発送製品）

- **掛川さん用にカスタマイズされたビュー**で1週間分のデータを取得
- このCSVは**GENへのインポートにもつながっている**（PSI転記とは別の業務フロー）

## 4. 在庫関連

### 4.1 フラグ管理

- 群馬通商Excelの**絞り込み機能**を使用（日付・フラグ）
- フラグは群馬通商Excel内で立てる。元データは**kintoneから引っ張ってくる**

### 4.2 仕損フラグの定義（新情報）

**事実**（出典: 掛川さんヒアリング）:
- 仕損 = **初期不良で再送**という意味
- **ECで買ったものがおかしくなっても仕損フラグになる**
- → 仕損は「販売チャネル問わず、初期不良による再送」

### 4.3 EC関連の識別子

**事実**（出典: 掛川さんヒアリング）:
- ECにはレコード番号がある（**これは全部にある**）
- **商品レコード番号は無い**
- **GEN受注番号はある**

### 確度更新

| 項目 | 変更前 | 変更後 | 理由 |
|------|--------|--------|------|
| 仕損品の定義 | D（推測） | **A**（初期不良の再送。EC含む） | 掛川さん明言 |
| EC区別方法 | D | **B**（GEN受注番号で区別できる可能性あり） | EC→GEN受注番号ありが判明 |
