## 1. 前提の整理

### 1.1 変えられないこと（制約）

| 制約 | 理由 | 確認セッション |
|------|------|--------------|
| GENはCSVインポートのみ（API連携不可） | GEN側の仕様制約 | S331（藤田さん確認） |
| kintone→GENの週次手動インポートは継続 | GEN制約により自動化不可 | S331 |
| kintone APIはNTT東日本基盤の制約あり | 当面CSV手動DL | S298, S317 |

### 1.2 変えられること（改善の余地）

| 改善経路 | 方法 | 確認セッション |
|---------|------|--------------|
| kintone側の作業最適化 | 宮崎さんに依頼して改善 | S331（藤田さん確認） |
| PSI Excel + 群馬通商Excelの代替 | 自前でプロトタイプ開発 | S313〜S323 |

### 1.3 作るもの

**PSI代替システム** = 群馬通商Excel + PSI Excelの機能を1つのシステムに統合

S326で定義したゴール:
> 掛川さんが毎週日曜に2時間かけているPSI Excel手動集計（年104h）を自動化する

対象範囲:
- 群馬通商Excelの入出庫管理（フラグ管理・レコード番号管理含む）
- PSI Excelの月別集計（受注・出荷・在庫）
- 上記の手動転記作業の自動化

## 2. 現状コストの試算

### 2.1 掛川さんの作業（事実ベース）

| 作業 | 頻度 | 所要時間 | 年間工数 | 確度 |
|------|------|---------|---------|------|
| PSI Excel手動集計 | 毎週日曜 | 2時間/回 | **104時間/年** | A（S326で確認） |
| kintone→GEN CSVインポート | 週次 | 不明（推定30〜45分） | 26〜39時間/年 | B（作業内容から推定） |
| 群馬通商Excel転記（kintone→Excel） | 週次 | 不明（推定45分〜1h） | 39〜52時間/年 | B（作業内容から推定） |

**注意**:
- PSI集計の2時間/回は確認済み事実（S326）
- kintone→GENインポートとExcel転記は作業内容からの推定。**掛川さんに実時間を確認すべき**
- kintone→GENインポートはGEN制約により自動化不可。PSI代替システムの効果には含めない

### 2.2 PSI代替システムで削減できるコスト

PSI代替で直接削減できるのは「PSI集計」と「群馬通商Excel転記」の部分。
kintone→GENインポートはGEN制約により対象外。

| 作業 | 現状 | PSI代替後 | 削減量 |
|------|------|----------|--------|
| PSI Excel手動集計 | 2h/週（104h/年） | 自動集計（ほぼゼロ） | **104h/年** |
| 群馬通商Excel転記 | 推定45分〜1h/週 | DB取込で自動化 | **39〜52h/年** |
| kintone→GEN CSVインポート | 推定30〜45分/週 | **変更なし（GEN制約）** | 0h/年 |
| **合計削減量** | | | **143〜156h/年** |

**人件費換算**（仮説: 時給3,000円として）:
- 143〜156時間 × 3,000円 = **43〜47万円/年**

### 2.3 間接効果（定量化困難だが重要）

| 効果 | 説明 | CoQ分類 |
|------|------|---------|
| 在庫情報の鮮度向上 | 週次Excel集計→DB即時反映。製販会議での判断精度向上 | 予防コスト（投資） |
| 仕損品の可視化 | バッテリ仕損24%の傾向分析が可能に。現状は追跡困難 | 内部失敗コスト削減 |
| 転記ミスの解消 | 手動転記（フラグ表記揺れ: 仕損/仕損品/不具合仕損）→コード化 | 内部失敗コスト削減 |
| 掛川さんの日曜作業解消 | 休日出勤相当の作業が平日自動処理に | 働き方改善 |
| データの一元化 | 群馬通商Excel+PSI Excelの二重管理解消 | 評価コスト削減 |

### 2.4 仕損品コストの試算（参考）

**事実**（S324b）: kintone出庫データ2日分で仕損品19件/78件（24%）、全てバッテリー

| 項目 | 推定値 | 確度 |
|------|--------|------|
| バッテリ単価 | 未確認（要GEN品目マスタ or 経理確認） | D |
| 月間仕損品件数 | 未確認（2日分データの外挿は危険） | D |

**仮説**: 仮にバッテリ単価5万円、月間仕損品30件とすると:
- 仕損品コスト = 5万円 × 30件 = **150万円/月**（再送分の材料費）
- PSI代替システムで仕損品自体は減らないが、**傾向分析→原因対策のサイクル**を回す基盤になる
- 業界事例（事例J）では不良率1年で半減の実績あり

**注意**: 上記は仮の数値。実際の仕損品コストは掛川さん・経理に確認が必要。

## 4. ROI試算

```
ROI = (年間便益 - 年間運用コスト) / 初期投資 × 100
投資回収期間 = 初期投資 / 年間純便益
```

### 4.1 Phase 1+2のみ（受注+出荷+在庫）

Phase 1+2で掛川さんの主要作業（PSI集計+群馬通商Excel転記）をカバー。

| 項目 | 最小 | 最大 |
|------|------|------|
| 初期投資 | 42万円 | 66万円 |
| 年間便益（工数削減） | 43万円 | 47万円 |
| 年間運用コスト | 6万円 | 12万円 |
| **年間純便益** | **31万円** | **41万円** |
| **ROI** | **47%** | **98%** |
| **投資回収期間** | **1.0年** | **2.1年** |

### 4.2 Phase 1+2+3（フルスコープ）

| 項目 | 最小 | 最大 |
|------|------|------|
| 初期投資 | 72万円 | 126万円 |
| 年間便益（工数削減+間接効果） | 50万円 | 70万円 |
| 年間運用コスト | 6万円 | 24万円 |
| **年間純便益** | **26万円** | **64万円** |
| **ROI** | **21%** | **89%** |
| **投資回収期間** | **1.1年** | **4.8年** |

**間接効果の加算**（Phase 3に7〜23万円/年を仮定）:
- 製販会議の質向上（判断精度→欠品・過剰在庫の削減）
- 仕損品傾向分析による改善サイクル
- ※定量化困難なため保守的に見積もり

### 4.3 ROI感度分析

ROIに最も影響する変数:

| 変数 | 影響度 | 現在の値 | 確認方法 |
|------|--------|---------|---------|
| PSI集計の実作業時間 | 大 | 2h/週（確度A） | 確認済み |
| 群馬通商Excel転記の実時間 | 中 | 推定45分〜1h/週（確度B） | 掛川さんヒアリング |
| 開発工数 | 大 | 240〜420h（確度B） | Phase 1着手で精度向上 |
| 仕損品の月間件数・単価 | 大（間接効果に影響） | 未確認（確度D） | 掛川さん+経理確認 |

## 6. 結論

### 定量的な結論

| 項目 | 値 |
|------|-----|
| 直接削減効果 | 143〜156時間/年（43〜47万円/年） |
| 投資額（Phase 1+2） | 42〜66万円 |
| 投資回収期間 | 1.0〜2.1年 |
| ROI | 47〜98% |

### 定性的な結論

1. **掛川さんの日曜2時間作業がゼロになる** — 最も分かりやすい効果
2. **仕損品の可視化基盤ができる** — 今は追跡困難、DB化で傾向分析可能に
3. **在庫情報の鮮度向上** — 週次Excel→DB即時反映、製販会議の質が上がる
4. **業界ROI水準と同等** — 投資として妥当な範囲
