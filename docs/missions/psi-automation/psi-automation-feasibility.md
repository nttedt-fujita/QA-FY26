## 4. 実現可能性の判断

### 結論: **実現可能。着手できる**

**根拠**:

1. **データソースの80%以上が特定済み** — 受注[A]、出荷[A]、群馬通商在庫[A]、拠点別在庫[A/B+]、生産[B]
2. **kintone APIでデータ取得可能** — CSVエクスポートだけでなくAPI経由も使える
3. **S313構想の骨格は有効** — 「kintone CSV → 小さなDB → 自動集計」は変わらない。インプットが1系統→3系統に増えただけ
4. **未確定のP-6（発注）は全体の一部** — 発注情報はバッテリのみ。他の品目にはない。後から追加できる

## 5. プロトタイプのスコープ案

### Phase 1: 最小プロト（受注+出荷の自動集計）

**目的**: kintone APIからデータを取得し、PSI相当の集計結果を出力できることを証明

**インプット**:
- kintone普及支援/受注管理 CSV → 受注数（品目別×月別）
- kintone発送製品(09) CSV → 出荷実績（品目別×週次）

**アウトプット**:
- 品目別×月別の受注・出荷集計（PSIの受注行・出荷行に相当）

**なぜこの範囲か**:
- 両方とも確度A（ソース確定済み）
- kintone CSV 2種のインポートだけで完結
- 掛川さんの年104h工数のうち、受注+出荷の集計が大きな割合を占める
- CSV手動DLは残るが、取り込み後の集計・転記作業が不要になる

### Phase 2: 在庫の追加

- kintone 71/72マスタ → 拠点別在庫
- 群馬通商Excel → 群馬通商在庫（初回取込 + 差分同期）

### Phase 3: 生産・発注・予測の追加

- 生産関連Excel → 生産数
- P-6（発注情報）→ 確認後追加
- 長屋さんの予測 → 手入力UIが必要
