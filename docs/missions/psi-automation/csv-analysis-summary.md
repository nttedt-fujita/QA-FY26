## 3. 各アプリで判明した重要事項

### 農業問合せ(19) — 391カラムの構造

- **製品種別①〜③**: 同一問合せで最大3製品を登録可能。各製品に対してシリアル×5、発生日、発生事象、代替品希望、住所等のフィールドが繰り返される
- **原因分類（130〜151列）**: チェックボックス形式で22種の原因を複数選択可能
  - ユーザー運用ミス / 製造ミス（組立工程） / 検査漏れ（受入検査・飛行検査後）
  - 設計不良（液剤タンク・カウル・基板） / バグ（ソフトウェア）
  - 製品不良（チャージャー・バッテリー・プロペラ・モーター・プロポ・粒剤散布装置）
  - 機体仕様 / 粒剤仕様 / 事象再現せず / 環境要因 / 代理店起因 / 対応中 / 不明
- **普及代筆（183〜391列）**: 代筆フォーム経由のデータ。バッテリ状態の詳細チェック項目（LED表示、端子状態等）が含まれる
- **仕損品提供有無** / **不良品回収有無** (177-178): 仕損品管理との連携

### 農業機体マスタ(71) — 107カラムの構造

- **構成管理情報（29〜50列）**: フライトコントローラ、プロポ、受信機、PDB/PCM/MC_Assy、Sig-HUB、LTE基板等の**部品構成をシリアルレベルで管理**
- **群馬通商関連（71〜76列）**: 入庫予定日、出庫ステータス、ロケ移動方法等
- **仕損品チェックボックス（80列）**: 仕損品フラグ
- **検収担当者（105〜107列）**: 受入検査の担当者情報

### 農業オプションマスタ(72) — 70カラムの構造

- **農業機体S/N（31列）**: 機体マスタとの紐づき（どの機体に付属するか）
- **次回充放電作業予定日（20列）**: バッテリ管理
- **群馬通商関連（45〜48列）**: 機体マスタと同様の構造
- **【削除予定】フィールド（17〜19列）**: 群馬通商関連の旧フィールドが残っている

### 発送(08) — 97カラムの構造

- **発送先情報が3セット**: 通常発送先、別送先？、さらに別送先？（46〜61列に発送先名・住所が3回出現）
- **出庫管理（35〜41列）**: 記入→入力→コメント送信→ピッキング→検品→ドライバー→出庫日の**作業ステップ管理**
- **仕損品決裁No. / 仕損品フラグ（76〜77列）**: 仕損品の決裁管理
- **保険金額（87列）**: 発送時の保険

### 発送製品(09) — 52カラムの構造

- **発送(08)の子レコード**: 発送レコード番号で親と紐づく
- **仕損品フラグ（32列）**: 個別製品の仕損品管理
- **出庫登録対象/対象外（34列）**: GENへの出庫登録制御
