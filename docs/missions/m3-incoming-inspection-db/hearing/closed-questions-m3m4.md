# クローズドクエスチョン（M3/M4）

**作成日**: 2026-03-05（Session 14）
**目的**: オープンな質問を「こういう認識でいいですか？」形式に変換し、ヒアリングを効率化

---

## 使い方

「はい/いいえ/修正あり」で答えられる形式にしている。
修正ありの場合は、どこが違うかを聞く。

## M3: 受入検査DB化

### カテゴリ A: 基本構造

| # | 提案 | 確認ポイント |
|---|------|-------------|
| A-1 | **シートは「メカ/エレキ/Api」の3分類で統一してよい？** | 他の分類方法があるか |

### カテゴリ B: 列構成の統一

現状: メカには「発注番号」があるがエレキ/Apiにはない

| # | 提案 | 確認ポイント |
|---|------|-------------|
| B-1 | **全シート共通で「発注番号」を必須項目にしてよい？** | エレキ/Apiでも使っているはず |
| B-2 | **「人工」列は不要では？検査工数(分)があれば計算できる** | 残す理由があるか |
| B-3 | **最終列は「品質基準書修正要否と箇所」で統一してよい？**（Apiだけ「その他」になっている） | 別の意味で使っているか |

### カテゴリ C: 入力ルール

現状: 検査数量の列に「20なみ検」「外観全数」「N-5」「西村」など混在

| # | 提案 | 確認ポイント |
|---|------|-------------|
| C-1 | **検査数量は「数値のみ」にして、検査方法は別列にする？** | |
| C-2 | **検査方法は選択式にする？**（例: 全数/抜取5/抜取20/外観のみ/員数のみ） | 他のパターンがあるか |
| C-3 | **作業者は選択式にする？**（マスタ登録制） | |
| C-4 | **サプライヤは選択式にする？**（マスタ登録制、表記揺れ防止） | |

### カテゴリ D: 不良記録

現状: 不良数量が空欄でも備考に不良内容が書いてある

| # | 提案 | 確認ポイント |
|---|------|-------------|
| D-1 | **不良があったら「不良数量」は必須入力にする？** | |
| D-2 | **「検査結果・備考」を分離する？**（判定/不良内容/対策/申し送りを別々に） | |
| D-3 | **「不問」判定の場合は理由を必須にする？** | |
| D-4 | **「要協議」「再発防止依頼」はステータス管理する？**（未対応/対応中/完了） | |

### カテゴリ E: 運用フロー

| # | 提案 | 確認ポイント |
|---|------|-------------|

## 共通: 分析・予防の観点

### カテゴリ I: 分析に必要な入力項目

「この分析がしたいなら、この項目を入力画面に追加する必要がある」という逆算。

| # | 質問 | 「はい」の場合 |
|---|------|---------------|
| I-1 | **ロット単位で品質を追跡したいですか？** | → 入力項目に「ロットID」を追加 |
| I-2 | **検査治具・検査場所別の分析は必要ですか？** | → 入力項目に「検査治具」「検査場所」を追加 |
| I-3 | **作業者の属性（身長・経験年数）を分析に使いたいですか？** | → 作業者マスタに属性を追加 |
| I-4 | **工程の詳細分類は必要ですか？**（「社内/ナカヨ/客先」より細かく） | → 工程マスタを設計 |
| I-5 | **不良内容の選択肢はこれで足りますか？**（インサート不良/キズ/異物混入/寸法NG/その他） | → 不足があれば追加 |

### カテゴリ J: 現在監視（リアルタイム）

| # | 質問 | 「はい」の場合 |
|---|------|---------------|
| J-1 | **当日の検査件数・不良件数をリアルタイムで見たいですか？** | → ダッシュボードに「本日」集計を追加 |
| J-2 | **管理図（SPC）で日々の不良率を監視したいですか？** | → QuickSightで管理図を設定 |
| J-3 | **管理図はどの単位で見たいですか？** | 日別 / ロット別 / 週別 / 月別 |

### カテゴリ K: 未来予測（早期警告）

| # | 質問 | 「はい」の場合 |
|---|------|---------------|
| K-1 | **不良率が悪化傾向にあるとき、自動で警告がほしいですか？** | → QuickSight異常検知を設定 |
| K-2 | **特定のサプライヤで不良が増えたとき、自動で気づきたいですか？** | → サプライヤ別の異常検知を設定 |
| K-3 | **「不良が起きる前に防ぐ」仕組みは必要ですか？** | → 管理図 + 異常検知の優先度を上げる |

## 補足: Excelレビューで見つけた問題点（根拠）

クローズドクエスチョンの背景として提示できる。

| 問題 | 具体例 |
|------|--------|
| 列構成の不統一 | メカには発注番号があるがエレキ/Apiにはない |
| 検査数量に複数情報 | 「20なみ検」「外観全数」「西村」が同じ列 |
| 作業者名の表記揺れ | 「西村・西浦」「西村　西浦」「西村西浦」（23回が5パターン） |
| サプライヤ名の表記揺れ | 「金森産業（株）」「金森産業株式会社」 |
| 不良数量が空欄 | 備考に「油漏れ8個確認」と書いてあるのに不良数量は空欄 |
| 備考に複数情報混在 | 判定/不良内容/対策/申し送りが1列に |
| 不具合発生シートに日付なし | いつ発生したか追跡できない |
