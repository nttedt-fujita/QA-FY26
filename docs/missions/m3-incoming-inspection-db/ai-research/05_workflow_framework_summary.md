# 作業フロー × フレームワーク 対応表

**現在地: Phase 1（一次調査前）**

---

## 全体の流れ

```
Phase 1 ──→ Phase 2 ──→ Phase 3 ──→ Phase 4 ──→ Phase 5
現状把握     記録基盤     分析・最適化   高度化      中長期改善
                │
                └─── Phase 3〜5 はすべて記録データの蓄積が前提 ───
```

---

## Phase 1: 現状把握 ← 今ここ

| やること | 使うフレームワーク | アウトプット |
|---|---|---|
| 300h の内訳を分解する | **CoQ** | 予防/評価/内部失敗/外部失敗の配分 |
| 検査プロセスの流れを描く | **VSM** | 工程フロー図（待ち時間・ボトルネック付き） |
| 「なぜ 300h か」の要因を洗い出す | **特性要因図** | 4M 分類された要因マップ |

---

## Phase 2: 記録基盤の構築

| やること | 使うフレームワーク | アウトプット |
|---|---|---|
| 記録すべきデータ項目を定義 | Phase 1 の結果から導出 | データ項目一覧 |
| 記録の仕組みを設計・構築 | — | DB / ツール / フォーマット |
| 運用を開始しデータを蓄積 | — | 検査記録データ |

**ここで「ツール制作」が初めて正当化される。目的は「Measure のためのデータ収集」。**

---

## Phase 3: データ分析・検査最適化

| やること | 使うフレームワーク | アウトプット |
|---|---|---|
| 工数が集中している部品・不良を特定 | **パレート図** | 重点改善対象リスト |
| 部品×不良モードのリスク優先順位付け | **FMEA** | RPN ランキング → 検査重点の再配分 |
| 抜取検査計画の妥当性を検証 | **ISO 2859 / JIS Z 9015** | 適正化されたサンプルサイズ |

---

## Phase 4: 検査プロセスの高度化

| やること | 使うフレームワーク | アウトプット |
|---|---|---|
| 検査員間の判定ばらつきを定量化 | **MSA / Gauge R&R** | 測定システムの信頼性評価 |
| AI 活用の検討 | — | 判定基準が明確な検査項目への AI 適用計画 |

**AI は Phase 4 で初めて登場する。Phase 1〜3 を飛ばして AI を入れても効果測定ができない。**

---

## Phase 5: 中長期的な検査負荷低減

| やること | 使うフレームワーク | アウトプット |
|---|---|---|
| サプライヤー品質の傾向を時系列で監視 | **SPC** | 管理図によるサプライヤー層別 |
| サプライヤーの工程能力向上を支援 | **SQM** | 監査・スコアカード・CAR の仕組み |
| データに基づき検査頻度を段階的に軽減 | SPC + SQM | Level 1→4 のサプライヤー格付け |

---

## 一枚で見る対応関係

```
フェーズ          フレームワーク              前提条件
─────────────────────────────────────────────────────────────
Phase 1 現状把握   CoQ / VSM / 特性要因図     現場ヒアリングで着手可
     │
Phase 2 記録基盤  （ツール/DB設計）           Phase 1 の結果
     │
Phase 3 分析      パレート / FMEA / 抜取計画  記録データの蓄積
     │
Phase 4 高度化    MSA / AI活用検討            判定基準の明確化
     │
Phase 5 中長期    SPC / SQM                  3〜6ヶ月のデータ蓄積
```
