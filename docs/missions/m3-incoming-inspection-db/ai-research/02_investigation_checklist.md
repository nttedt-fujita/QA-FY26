# 一次調査チェックリスト

**目的**: Define フェーズを確定し、Measure フェーズで「何を記録すべきか」を決めるための情報収集

---

## A. 工数の内訳把握（300h の分解）

- [ ] 検査対象の部品種類数
- [ ] 月間の受け入れロット数
- [ ] 1 ロットあたりの平均検査時間
- [ ] 検査の種類別の時間配分（目視 / 寸法測定 / 機能試験 / 書類確認 など）
- [ ] 検査以外の付帯作業時間（搬入・搬出・記録・連絡・待ち時間）
- [ ] 不合格時の再検査・返品処理にかかる時間
- [ ] 検査員の人数と 1 人あたりの月間工数

---

## B. 検査プロセスの詳細

- [ ] 検査手順書・検査基準書は存在するか
- [ ] 抜取検査計画の根拠（AQL 値、サンプルサイズの決め方）
- [ ] 使用している規格（JIS Z 9015 / ISO 2859 など）
- [ ] 検査に使用する機器・設備
- [ ] 判定基準は明文化されているか、暗黙知に依存しているか
- [ ] 検査員による判定のばらつきはあるか（感覚値でも）
- [ ] 検査のボトルネックはどこか（現場の肌感覚）

---

## C. 記録の現状

- [ ] 現在どんな記録が取られているか（紙 / Excel / システム など）
- [ ] 記録されている項目は何か
- [ ] 記録されていないが必要だと感じている項目はあるか
- [ ] 過去の記録から不良傾向を分析したことはあるか
- [ ] 記録が取れていない理由（時間がない / フォーマットがない / 優先度が低い）

---

## D. 品質実績

- [ ] 月間の不合格率（全体 / 部品種類別 / サプライヤー別）
- [ ] 主な不良モード（寸法不良 / 外観不良 / 機能不良 / 材料不良 など）
- [ ] 不合格品の処置フロー（返品 / 特別採用 / 廃棄）
- [ ] 後工程（製造ライン）での品質クレーム発生状況
- [ ] 市場・顧客からの品質クレームで受け入れ検査起因のものはあるか

---

## E. サプライヤー管理

- [ ] サプライヤーの数
- [ ] サプライヤーの選定・評価基準はあるか
- [ ] サプライヤーごとの品質実績の差は把握されているか
- [ ] サプライヤー監査は実施しているか
- [ ] サプライヤーからの検査成績書・出荷検査データは受領しているか
- [ ] サプライヤーの工程能力（Cpk 等）は把握しているか

---

## F. 既存の改善の取り組み

- [ ] 過去に検査プロセスの改善を試みたことはあるか
- [ ] AI 活用の具体的な検討状況（橋梁点検 AI の技術内容・転用可能性）
- [ ] DB 化の検討状況（要件 / ツール選定 / 担当）
- [ ] 他部門で使っている品質管理の仕組みはあるか

---

## 調査方法の推奨

| 方法 | 対象 | 目的 |
|------|------|------|
| 現場観察 | 検査作業の実際の流れ | VSM 作成のためのプロセス把握 |
| ヒアリング | 検査担当者 | 工数内訳・ボトルネック・暗黙知の把握 |
| ヒアリング | SCM 管理者 | 目標値・優先度・制約条件の確認 |
| ヒアリング | 製造ライン担当 | 後工程での品質問題の実態把握 |
| 書類確認 | 検査基準書・手順書・過去記録 | 現行ルールと実態の乖離確認 |
