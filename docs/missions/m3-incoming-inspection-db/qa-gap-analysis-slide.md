# 品質管理視点のギャップ分析

## 現行Excel vs 理想的IQC

---

## 工程全体の流れ

```
サプライヤ → [IQC 受入検査(M3)] → 製造工程 → [PQC 工程内検査(M4)] → OQC → 顧客
```

- **M3 = IQC（受入検査）**: 入荷部品の品質確認
- **M4 = PQC（工程内検査）**: 製造中の不良検出

→ **両者は「部品」で紐づく**（ロットIDでトレース可能が理想）

---

## 現行Excel（As-Is）

### ✅ できていること

- 検査記録（品名、担当者、工数）
- 結果（OK/NG/返品）
- カテゴリ分類（メカ/エレキ/Api）
- 月別集計（シート分割）

---

## 現行Excel（As-Is）続き

### ❌ 欠けていること（ギャップ）

| 欠如項目 | 影響 |
|----------|------|
| ロット番号なし | トレースバック不可 |
| サプライヤIDなし | 供給元追跡不可 |
| 発注番号なし | 入荷追跡不可 |
| AQL基準なし | 抜取検査の根拠なし |
| 8Dフローなし | 改善サイクル未確立 |
| M4との紐づきなし | 横断分析不可 |

---

## 現行Excel（As-Is）続き

### ⚠️ データ品質の問題

- 杉山さんの記録のみ（一部データ）
- 未来日付10件、入荷日空欄34件
- 工数未記入約80件

→ **記録の網羅性・正確性に課題**

---

## 理想的なIQCシステム（To-Be）

### トレーサビリティ

- ロット番号で入荷単位を管理
- サプライヤID + 発注番号で追跡
- 製品→原料（トレースバック）可能

### 検査基準

- AQL基準（品目別に設定）
- 全数 vs 抜取の使い分け
- 明確な合否判定基準

---

## 理想的なIQCシステム（To-Be）続き

### 8Dフロー（問題解決）

1. **不良発生** → 記録（5W2H）
2. **原因分析** → 4M（人・機械・方法・材料）
3. **対策** → 暫定/恒久
4. **効果確認** → 不良率推移

### M3/M4連携

- 共通の部品マスタ
- ロットIDで受入→工程を追跡

---

## 導入の制約

| 制約 | 詳細 |
|------|------|
| 品質協定書 | 未締結（ナカヨ、渡辺製作所）→ M4データ収集に制約 |
| 段階的導入 | Phase1（分析）→ Phase2（入力デジタル化）→ Phase3（M3/M4統合） |

---

## 今できること（Phase 1）

### 分析・可視化（現行データの活用）

- 月別/品目別の不良傾向
- 工数分析
- パレート分析

→ **ヒアリング項目の深堀りに活用**
