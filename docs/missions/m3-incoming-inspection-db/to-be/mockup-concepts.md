## M3: 受入検査DB 画面構成

### 画面一覧（最小構成）

| # | 画面名 | 用途 | 優先度 |
|---|--------|------|--------|
| 1 | **検査入力画面** | 検査完了時に入力 | ★★★ |
| 2 | 検査一覧画面 | 過去の検査記録を検索・閲覧 | ★★ |
| 3 | サプライヤ別集計 | どのサプライヤで不良が多いか | ★ |

### 画面1: 検査入力画面（詳細）

```
┌─────────────────────────────────────────┐
│ 受入検査 入力                            │
├─────────────────────────────────────────┤
│                                         │
│ 入荷日:      [2026-03-05]              │
│ 発注番号:    [______________]           │
│ 品番:        [選択 ▼] または [スキャン]  │
│ 品名:        (自動表示)                  │
│ サプライヤ:  (自動表示)                  │
│                                         │
│ 入荷数量:    [____]                     │
│ 検査数量:    [____]                     │
│ 検査方法:    [全数 ▼] [抜取5 ▼]        │
│                                         │
│ ─────────── 判定 ───────────            │
│ 結果:        ○合格  ○不合格  ○保留    │
│ 不良数量:    [____] (不合格時のみ)       │
│ 不良内容:    [選択 ▼] (複数選択可)       │
│              □インサート不良            │
│              □キズ                      │
│              □異物混入                  │
│              □寸法NG                    │
│              □その他: [________]        │
│                                         │
│ 写真:        [撮影] (不良時のみ)         │
│                                         │
│ ─────────── 備考 ───────────            │
│ 対策・申し送り: [_______________]       │
│ 基準書修正要否: ○不要 ○要 [理由____]   │
│                                         │
│ 作業者:      [西村 ▼]                   │
│ 検査工数:    [__] 分                    │
│                                         │
│         [下書き保存]  [登録]            │
└─────────────────────────────────────────┘
```

**ポイント**:
- 品番選択で品名・サプライヤが自動表示（マスタ連携）
- 不良内容は選択式（表記揺れ防止）
- 判定に応じて入力項目が変わる（条件分岐）

## M4: 工程不良DB 画面構成

### 画面一覧（最小構成）

| # | 画面名 | 用途 | 優先度 |
|---|--------|------|--------|
| 1 | **不良入力画面** | 不良発見時に入力 | ★★★ |
| 2 | 不良一覧画面 | 過去の不良を検索・閲覧 | ★★ |
| 3 | 対策ステータス管理 | 未対応/対応中/完了 | ★★ |
| 4 | パレート図 | どの不良が多いか可視化 | ★ |

### 画面1: 不良入力画面（詳細）

```
┌─────────────────────────────────────────┐

│ 品名:        (自動表示)                  │
│ サプライヤ:  (自動表示)                  │
│                                         │
│ 不良数量:    [____]                     │

│                                         │

│                                         │
```

- 発生場所（社内/ナカヨ/客先）を選択式に
- 受入検査DBとの紐付け（品番・サプライヤ共通）
- 対策ステータスで追跡可能に
