## 1. QA分析のフレームワーク解説

### 1.1 パレート図とは

**目的**: 「どこに集中して対策すべきか」を可視化

```
不良件数
│
│ ████████████                             ─┬─ 80%ライン
│ ██████████                                │
│ ████████                                  │
│ ██████                                    │
│ ████                                      │
│ ██                                        │
│ █                                         │
└──────────────────────────────────────────→
  インサート キズ  異物 寸法NG その他
   不良
```

**分析できること**:
| 分析の切り口 | 例 |
|-------------|-----|
| **不良の種類別** | インサート不良が全体の40%を占める → インサート対策が最優先 |
| **サプライヤ別** | 金森産業が全体の50%を占める → 金森との品質改善が優先 |
| **工程別** | 成型工程が60%を占める → 成型の管理を強化 |
| **部品別** | タンクが30%を占める → タンクの検査強化 |

**ポイント**: 「20%の原因が80%の問題を起こしている」（80:20の法則）を可視化

---

### 1.2 管理図（SPC）とは

**目的**: 「工程が安定しているか」「異常が発生していないか」を監視

```
        UCL（上方管理限界）
        ─────────────────────●───  ← 異常！
                           ○
                      ○         ○
        CL（中心線）───○───○───○───○────
               ○        ○
        ─────────────────────────────────
        LCL（下方管理限界）

        ↑時間軸（日、週、ロット）
```

**分析できること**:
| パターン | 意味 |
|---------|------|
| 点がUCL/LCLを超えた | **異常発生**（対策必要） |
| 点が連続して上昇/下降 | **傾向あり**（悪化の兆候） |
| 点がCL付近で安定 | **工程は安定** |

---

### 1.3 トレンド分析とは

**目的**: 「時間とともに良くなっているか/悪くなっているか」を把握

```
不良率
│
│                    ●
│               ●
│          ●
│     ●
│ ●
└────────────────────→
  1月  2月  3月  4月  5月
```

**分析できること**:
- 対策後に不良率が下がったか
- 季節変動があるか
- 傾向的に悪化していないか

## 2. 自作すべきもの vs QuickSightに任せるもの

Session 9のQuickSight調査を元に整理。

### 2.1 仕分けの考え方

| 区分 | 自作すべき | QuickSightに任せる |
|------|-----------|-------------------|
| **入力系** | ✅ | - |
| **分析系** | - | ✅ |
| **一覧・検索** | ✅ or △ | △ |

**理由**:
- 入力画面は**業務フローに密結合** → 使い勝手が重要 → 自作
- 分析は**グラフ作成が大変** → 既製品に任せる → QuickSight
- 一覧・検索は**両方で可能** → どちらでも

---

### 2.2 具体的な仕分け

#### 自作すべきもの（入力系）

| 機能 | 理由 |
|------|------|
| **受入検査 入力画面** | 業務フローに合わせたUI必須 |
| **工程不良 入力画面** | 現場で素早く入力できる必要 |
| **マスタ管理**（品番、サプライヤ、作業者） | 選択肢の管理 |
| **ステータス管理**（未対応→対応中→完了） | ワークフロー |

#### QuickSightに任せるもの（分析系）

| 機能 | QuickSightでできる |
|------|-------------------|
| **パレート図** | ✅ 棒グラフ + ソート |
| **管理図（SPC）** | ✅ 計算フィールド + 参照線 |
| **トレンド分析** | ✅ 折れ線グラフ |
| **サプライヤ別集計** | ✅ ピボットテーブル |
| **部品別集計** | ✅ ピボットテーブル |
| **フィルター**（期間、カテゴリ） | ✅ 標準機能 |
| **ドリルダウン** | ✅ 標準機能 |
| **異常検知** | ✅ AI機能 |

#### どちらでも（一覧・検索）

| 機能 | 自作 | QuickSight |
|------|------|-----------|
| **検査記録一覧** | ○ | ○ |
| **検索・絞り込み** | ○ | ○ |
| **CSVエクスポート** | △ | ○ |

## 3. モックアップへの反映

### 3.1 モックアップで見せるべきもの（自作する部分）

| 画面 | 見せる意味 |
|------|-----------|
| **受入検査 入力画面** | ✅ 必須（業務フロー確認） |
| **工程不良 入力画面** | ✅ 必須（業務フロー確認） |

| 画面 | 理由 |
|------|------|

| トレンド分析 | 同上 |
