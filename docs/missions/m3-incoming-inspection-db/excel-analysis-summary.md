# 受入検査Excel分析サマリー

**作成日**: 2026-03-06 (Session 24)
**更新日**: 2026-03-06 (Session 26) — 前処理の記載追加
**対象データ**: 受入検査作業集計.xlsx（杉山さんの記録のみ）
**対象期間**: 2024-09 〜 2026-03（約18ヶ月）

---

## 1. 分析の前処理

Excelデータをそのまま集計すると精度が低いため、以下の前処理を実施した。

| 処理 | 内容 | 効果 |
|------|------|------|
| 名寄せ | 表記揺れの統一（品名18パターン、検査内容39パターン） | 集計精度向上 |
| 日付修正 | 2026年→2025年の誤入力を修正（10件） | 月別集計の正確化 |
| 入荷日補完 | 入荷日空欄→検査完了日で代用（26件） | 集計対象の拡大 |

処理スクリプト: [tools/incoming_inspection/](../../../tools/incoming_inspection/)

---

## 2. データの範囲と限界

### 重要な前提

| 項目 | 事実 |
|------|------|
| **記録者** | 杉山さんのみ（他の作業者は紙で記録） |
| **総レコード数** | 574件 |
| **全期間合計工数** | 530.5時間 |
| **概算との乖離** | 月300時間と聞いていたが、このExcelでは最大67時間/月 |

**結論**: このExcelは受入検査全体の一部のみを記録している。

### データ品質の問題

| 問題 | 件数 | 詳細 | 対応 |
|------|------|------|------|
| **未来日付（2026-11/12）** | 10件 | 2025年の誤入力 | 前処理で修正済み |
| **入荷日空欄** | 34件 | 社内作業との混在 | 26件は検査完了日で補完 |
| **工数未記入** | 約80件 | 検査対象外、入荷記録のみ、記入漏れ | 集計から除外 |
| **矢印記号（↓↑）** | 15件 | 「前後と同じ」の意味だが集計されていない | 未対応 |
| **作業者も空欄** | 約30件 | 入荷記録のみで検査未実施 | 集計から除外 |

**課題**: 記録が適切にできていない。確認できる体制も整っていない。

## 3. 分析結果（Excelから分かったこと）

### 月別傾向

| 期間 | 傾向 |
|------|------|
| 2024-09〜10 | 立ち上げ期（8〜26時間/月） |
| 2024-11〜12 | 増加（39〜53時間/月） |
| 2025-01〜08 | 変動あり（8〜32時間/月） |
| 2025-12〜2026-02 | **急増（56〜67時間/月）** |

### 工数上位品目

| 品目 | 工数 | 検査内容 |
|------|------|---------|
| PCM Assy | 24時間/件 | 外観+通電 |
| チャージャ | 16.7時間/件 | 動作確認 |
| バッテリ LOG抜き | 累計13.5時間 | ログ抜き |
| FA28.2*9.2 (プロペラ) | 累計約30時間 | 外観+キャリブレーション |
| U8 Lite Armset | 累計約25時間 | 外観+キャリブレーション |

### 作業者別傾向

| 作業者 | 期間 | 特徴 |
|--------|------|------|
| 西村 | 2024-09〜2025-11 | メイン担当、平均30分/件 |

| 角田 | 随時 | 複雑案件（PCM等）担当 |

### 検査内容の傾向

| 検査内容 | 平均工数/件 |
|----------|------------|
| 外観のみ | 34分 |
