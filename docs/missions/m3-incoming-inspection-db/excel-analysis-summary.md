# 受入検査Excel分析サマリー

**作成日**: 2026-03-06 (Session 24)
**更新日**: 2026-03-06 (Session 30) — 構造的な問題を追記
**対象データ**: 受入検査作業集計.xlsx（杉山さんの記録のみ）
**対象期間**: 2024-09 〜 2026-03（約18ヶ月）

---

## 1. 分析の前処理

Excelデータをそのまま集計すると精度が低いため、以下の前処理を実施した。

| 処理 | 内容 | 効果 |
|------|------|------|
| 名寄せ | 表記揺れの統一（品名18パターン、検査内容39パターン） | 集計精度向上 |
| 日付修正 | 2026年→2025年の誤入力を修正（10件） | 月別集計の正確化 |
| 入荷日補完 | 入荷日空欄→検査完了日で代用（26件） | 集計対象の拡大 |

処理スクリプト: [tools/incoming_inspection/](../../../tools/incoming_inspection/)

## 2. データの範囲と限界

### 重要な前提

| 項目 | 事実 |
|------|------|
| **記録者** | 杉山さんのみ（他の作業者は紙で記録） |
| **総レコード数** | 574件 |
| **全期間合計工数** | 530.5時間 |
| **概算との乖離** | 月300時間と聞いていたが、このExcelでは最大67時間/月 |

**結論**: このExcelは受入検査全体の一部のみを記録している。

### データ品質の問題

| 問題 | 件数 | 詳細 | 対応 |
|------|------|------|------|
| **未来日付（2026-11/12）** | 10件 | 2025年の誤入力 | 前処理で修正済み |
| **入荷日空欄** | 34件 | 社内作業との混在 | 26件は検査完了日で補完 |
| **工数未記入** | 約80件 | 検査対象外、入荷記録のみ、記入漏れ | 集計から除外 |
| **矢印記号（↓↑）** | 15件 | 「前後と同じ」の意味だが集計されていない | 前処理で修正済み（Session 29） |
| **作業者も空欄** | 約30件 | 入荷記録のみで検査未実施 | 集計から除外 |

**課題**: 記録が適切にできていない。確認できる体制も整っていない。

### 構造的な問題（前処理では解決できない）

| 問題 | 詳細 | 影響 |
|------|------|------|
| **不良数量の記録が不完全** | 空欄でも備考に不良記述あり、「保留」「1個不足」など数値以外の値 | 不良品数を正確にカウントできない |
| **検査基準書の不足** | エレキ「検査基準書は無い、図面が頼り」、Api「基準書の作成要」 | 判定基準が不明確 |
| **判定基準が属人的** | 「個人主観」「不問にしたい」「要協議」など | 合否判定にばらつき |
| **「良」判定に懸念事項が埋もれている** | 「良　ピン跡に黒ずみ（不問）」など | 潜在的な品質問題を追跡できない |
| **検査結果・備考列に情報が混在** | 合否判定、不良内容、対策、申し送りが1列に | 分類・集計が困難 |
| **不具合発生シートとの紐づきなし** | 日付・数量・対策ステータスもなし | 不良の追跡・改善ができない |
| **列構成の不統一** | メカ/エレキ/Apiで列が異なる（発注番号の有無など） | シート統合時にマッピング必要 |
| **検査数量の表記が87パターンに分散** | 「20」「なみ検20」「外観全数寸法抜き取り20」「N-5」など | 集計不可能 |

**結論**: このExcelは「記録」としては機能しているが、「分析」「改善」には使えない構造になっている。
