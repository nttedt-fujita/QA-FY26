## シート別レビュー

### 1. 検品結果報告（メカ）

**ヘッダー（5行目）:**
入荷日 / 検査完了日 / 発注番号 / 品番 / 品名 / サプライヤ / 入荷数量 / 不良数量 / 検査内容 / 検査資料 / 作業者 / 検査数量 / 検査工数(分) / 人工 / 検査場所 / 検査治具 / 検査結果・備考 / 品質基準書修正要否と箇所

**事実:**
- 251件のデータ。AC102（AirGrow）の受入検査が対象
- 1〜4行目にタイトル・注記あり（「SCMより渡される検査票と品質基準書を用意し作業をする」）
- 品質基準書の参照: 「2024品質基準書（AC102）A25-3000-0」「2024品質基準書(P13)P13-3000-0」
- 日付はExcelシリアル値（例: 45538 = 2024年9月頃）

**不良が記録された行の例:**
- 行27: 不良数=1「インサート逆入れあり、代品入庫済」（品番:EX000934-02, 吐出口, 金森産業）
- 行68: 不良数=13「不良多し、インサート抜け、吐出口削りすぎ」
- 行76: 不良数=4「シール内にゴミ4枚、後ろのシール右上がり傾向」
- 行91: 不良数=2「2点レール端面にキズあり、製造中による？」
- 行103: 不良数=4「樹脂のバリ、インサートなどチェック。不具合は交換依頼」
- 行114: 不良数=3「インサート抜け、カットライン不良」
- 行126: 不良数=14「汚れ、曇りキズ多し」（T-motor）
- 行178: 不良数=3「異物混入、成型時エア混入」
- 行210: 不良数=0 備考「初回納入は、全数寸法NG。メーカ側の温度管理の為マイナス寸法で出来上がり。作り直して再納入分」

**「良」判定だが注意事項が記載されている例:**
- 行16:「良　ピン跡に黒ずみ（不問）」
- 行23:「良　バリとパッキン面が同じ所にある」
- 行26:「良　インサート周辺にマステの断片あり(除去)」
- 行33:「良ネジ穴キツめの所あり深さ8ミリ以上」
- 行60:「良　ノギスでの寸法計測に難しさがある」
- 行63〜67:「良　ノギスで測ると設計値とずれます」（5件連続で同じ問題）
- 行131:「樹脂の中に小さい黒点あり（機能部品なので見た目は不問にしたい）」
- 行132:「バリの程度が気になる（許容範囲が不明なため、老衰の原因にならないかどうか）」

**課題:**
- 「良」判定の中に実質的な懸念事項が自由記述で埋もれている
- 「不問」「不問にしたい」「要協議」「メーカーに確認要」など、判断保留の記述がある
- 不良数量が空欄のまま、備考に不良の記述がある行が多数存在
- 不良数量に「保留」（行130）など数値でない値が入っている

## 横断的な課題

### A. 列構成の不統一

3つの検品結果報告シート（メカ/エレキ/Api）で列構成が微妙に異なる。

| 項目 | メカ | エレキ | Api |
|------|------|--------|-----|
| 発注番号 | C列にあり | **なし** | **なし** |
| 品番 | D列 | C列 | C列 |
| 品名 | E列 | D列 | D列 |
| 以降の列 | 1列ずれ | — | — |
| 最終列名 | 品質基準書修正要否と箇所 | 品質基準書修正要否と箇所 | **その他** |
| 人工列 | あり(N列) | あり(M列) | **なし** |

→ シートを統合した集計や、DB化する際に列のマッピングが必要

### B. 検査数量の表記が統一されていない

検査数量の列に、以下のような表記が混在している（全87パターン中の例）:

**数値のみ:** 「5」「20」「100」
**検査方法+数量:** 「20なみ検」「なみ検査　20個のみ」「外観全数,寸法20個」「外観全数　寸法抜き取り20個」「外観全数寸法なみ検20」
**検査方法のみ:** 「全数」「外観全数」「外観のみ」「員数のみ」
**抜き取り表記:** 「N-1」「N-5」「n-10」「n-20」「抜き取り5」「抜き取り５個」「抜き取り２０個」
**矢印（上の行と同じの意味?）:** 「↑」「↓」
**作業者名が入っている:** 「西村」「杉山」「中井・西村」
**その他:** 「?」「ー」「基準書無しなので5個」「131.5h」「2個/1箱」「6個/80分」

→ **検査方法、検査数量、作業者が1つの列に混在**している。集計不可能

### C. 作業者名の表記揺れ

同じ人物の組み合わせが複数の表記で記録されている:

| 表記パターン | 出現回数 | 同一の組み合わせ |
|-------------|---------|----------------|
| 西村・西浦 | 13回 | 同一 |
| 西村　西浦 | 3回 | 同一 |
| 西村西浦 | 4回 | 同一 |
| 西浦・西村 | 2回 | 同一（順番が逆） |
| 西浦　西村 | 1回 | 同一（順番が逆） |

→ 合計23回の記録が5パターンに分散。作業者別の集計ができない

| 作業者 | 回数 |

### E. 不良数量の記録が不完全

- 不良数量の列が**空欄でも、備考に不良の記述がある**行が多数
  - 例（メカ行23）: 不良数=空欄, 備考=「良　バリとパッキン面が同じ所にある」
  - 例（Api行24）: 不良数=空欄, 備考=「油漏れ8個確認社内報告済」
- 不良数量に**数値でない値**が入っている
  - 「保留」（メカ行130）
  - 「1個不足」（Api行54）
  - 「0」（不良なしの明示）と「空欄」（不良なし?未記入?）が混在
- **「不問」判定の基準が曖昧**
  - 「ピン跡に黒ずみ（不問）」「押し出しに黒済み（不問）」「機能部品なので見た目は不問にしたい」
  - → 不良ではあるが不問とした、その判断基準が記録されていない

### F. 「検査結果・備考」列に複数の情報が混在

1つの列に以下の情報がすべて入っている:

| 情報の種類 | 例 |
|-----------|-----|
| 合否判定 | 「良」「良好」「問題なし」 |
| 不良の詳細 | 「インサート逆入れあり」「キズ、リベット」 |
| 対処内容 | 「代品入庫済」「除去」「張り直し対応済」 |
| 測定結果 | 「板厚1.71～1.73㎜」「外径6.23～6.26」 |
| 検査方法のメモ | 「寸法は実際に入るネジを入れて確認」 |
| 判断の保留 | 「要協議」「メーカーに確認要」「再発防止依頼要」 |
| 基準書への要望 | 「インサートのサイズ、個数を基準書に入れてほしい」 |
| 作業の備忘録 | 「20本/袋詰め作業含む」「箱入り箱出し含む」 |

→ **合否判定、不良内容、対策、申し送り事項を分離できない**

### G. 検査基準書の不足

各シートの記述から、検査基準書が整備されていない部品が複数ある:

- エレキシート2行目:「検査基準書は無いため、整備が必要　現在は図面が頼り。」
- エレキ行25:「検査基準書無し」（ギヤBOX）
- エレキ行26:「基準書無し、ハーネス図面」（シャッターサーボ）
- Api行100:「基準書の用意必要」
- Api行101:「基準書の作成要」
- 備考等シート:「買い物部類の検査はどうする？必要であれば検査基準書の作成をしてほしい」

### H. 不具合発生シートの情報不足

受入検査シートと不具合発生シートの間で:
- **紐付け手段がない**（共通キーがない）
- 不具合発生シートに**日付・数量・対策ステータスがない**
- 不具合が「いつ」「何件」「どう対処されたか」を追跡できない
