# 用語の整理: QAとQC

| | QA（品質保証） | QC（品質管理） |
|---|---|---|
| **英語** | Quality Assurance | Quality Control |
| **役割** | 品質を**保証する仕組み**を作る | 工程内で品質問題を**検出・防止**する |
| **視点** | ユーザー・顧客視点 | 現場の作り手視点 |
| **範囲** | 全ライフサイクル | 製造工程が中心 |
| **具体例** | 評価手法の策定、検証基準の設定 | 受入検査の実施、工程内検査 |

> **ポイント**: QCはQAの中の一部。QAのほうが広い概念。

# 各段階の説明

| 略語 | 名称 | 意味 | 自社での内容 |
|------|------|------|-------------|
| **SQM** | Supplier Quality Management | サプライヤ品質管理 | サプライヤの選定・評価・監査 |
| **IQC** | Incoming Quality Control | 受入品質管理 | 入荷部品の検査（AQL抜取等） |
| **IPQC** | In-Process Quality Control | 工程品質管理 | 組立工程中の検査 |
| **OQC** | Outgoing Quality Control | 出荷品質管理 | 完成機の最終検査・飛行テスト |

> 自社は**部品を購入して組み立てるメーカー**なので、
> SQM（サプライヤの品質管理）が特に重要。

# 中国サプライヤの特有課題: サイレントチェンジ

**サイレントチェンジ** = サプライヤが無断で仕様を変更すること

| リスク | 影響 | 対策 |
|--------|------|------|
| バッテリーセルの材料変更 | 発火・膨張リスク | 定期分解検査 |
| FWの無断アップデート | 通信異常 | **M3でバージョン記録・自動検出** |
| 基板部品の代替品使用 | 性能劣化 | 電気特性測定 |

> **現状**: PO発行時にバージョン連絡を必須化（いしかわ担当）
> **強化案**: M3（受入検査DB）でFW/HWバージョンを自動記録・変更検出

# M3/M4の具体像: Excel管理からの脱却

| | 現状（Excel） | 目標（DB + アプリ） |
|---|---|---|
| **記録方法** | PC上のExcelファイル | タブレットで入力可能なアプリ |
| **検索** | ファイルを開いて検索 | 即座にクエリ・フィルタ |
| **分析** | 手作業で集計 | 自動パレート分析・傾向管理 |
| **共有** | ファイル共有 | ダッシュボードで品質G全員が閲覧 |
| **トレーサビリティ** | 限定的 | lot_idで受入〜工程を紐付け |

> **M3とM4はlot_idで連携**
> → 受入時に合格した部品が工程で不良 → サプライヤにフィードバック

# 優先順位の提案

## M3/M4（DB化）を先行、M1は並行で調査開始

**理由**:
1. **ソフトスキルが直接活きる** — DB設計は既存スキルそのまま
2. **早期に成果が見える** — DBが動けば品質G全員が使える
3. **他メンバーへの貢献** — いしかわさん・こいたばしさんのデータも集約
4. **M1は調査フェーズから** — 規格学習・評価項目選定は並行で可能
