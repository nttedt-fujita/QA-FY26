## 1. SPC（統計的工程管理）とは

### 1.1 一言で言うと

**「不良品が出る前に、工程の異常を検知する」手法**

### 1.2 従来の品質管理 vs SPC

```
【従来】
  製造 → 検査 → 不良品発見 → 対処（後手）
                 ^^^^^^^^
                 もう遅い

【SPC】
  製造 → データ収集 → 傾向分析 → 異常の兆候を検知 → 対処（先手）
                                ^^^^^^^^^^^^^^^^
                                不良品が出る前に気づく
```

### 1.3 具体例

例えば、ある部品の寸法を毎日測定しているとする：

```
日付    寸法(mm)
月曜    10.02
火曜    10.01
水曜    10.03
木曜    10.05  ← ちょっと上がってきた？
金曜    10.08  ← さらに上がった
土曜    10.12  ← 規格ギリギリ（10.15が上限）
```

SPCを使っていれば、**木曜の時点で「傾向がおかしい」と気づける**。
使っていなければ、**土曜に規格外が出てから慌てる**。

### 1.4 SPCの価値

| 観点 | 従来 | SPC |
|------|------|-----|
| 検知タイミング | 不良発生後 | 不良発生前 |
| 対応 | 後手（火消し） | 先手（予防） |
| コスト | 不良品廃棄・手直し | 予防で抑制 |
| データ活用 | 記録のみ | 傾向分析・予測 |

## 2. 管理図とは

### 2.1 一言で言うと

**「データの傾向をグラフで見える化して、異常を早期発見するツール」**

### 2.2 管理図の構成要素

```
     UCL(上方管理限界) ─────────────────────
                           ●
                      ●         ●
     CL (中心線)  ────●───●───●───●────●────
                 ●         ●

     LCL(下方管理限界) ─────────────────────

     ─────────────────────────────────────→ 時間
```

| 要素 | 英語 | 意味 |
|------|------|------|
| UCL | Upper Control Limit | 上方管理限界線（平均 + 3σ） |
| CL | Center Line | 中心線（平均値） |
| LCL | Lower Control Limit | 下方管理限界線（平均 - 3σ） |

### 2.3 管理限界の計算方法

```
UCL = 平均値 + (3 × 標準偏差)
CL  = 平均値
LCL = 平均値 - (3 × 標準偏差)
```

**3σ（シグマ）ルール**: 正規分布では、データの99.73%が平均±3σの範囲に収まる。この範囲を超えたら「異常」と判断する。

### 2.4 異常のパターン

管理図で「異常」と判断するルール：

| パターン | 状態 | 意味 |
|---------|------|------|
| 点がUCL/LCLを超えた | 明らかな異常 | 即座に対処が必要 |
| 7点連続で上昇（または下降） | トレンド | 傾向がある（ドリフト） |
| 7点連続で中心線の片側 | シフト | 工程が偏っている |
| 点が中心線付近に集中しすぎ | 層別不足 | データの取り方を見直す |

**図解**:

```
【正常】                    【異常：トレンドあり】
     ─── UCL                     ─── UCL
        ●   ●                           ●
    ●     ●   ●                       ●
     ─── CL                      ─── CL ●
      ●       ●                      ●
        ●                          ●
     ─── LCL                     ─── LCL


【異常：点が限界超え】       【異常：シフト（片側に偏り）】
     ─── UCL     ●←超えた        ─── UCL
        ●   ●                    ●  ●  ●  ●
    ●                             ●     ●
     ─── CL                      ─── CL
      ●       ●
        ●
     ─── LCL                     ─── LCL
```

## 3. パレート図とは

### 3.1 一言で言うと

**「どの問題が一番多いか、優先順位をつけるグラフ」**

### 3.2 パレート図の構成

```
件数                                      累積%
50 |████████                              100%
   |████████                         ─────
40 |████████ ████████                     80%
   |████████ ████████              ──
30 |████████ ████████ ████████            60%
   |████████ ████████ ████████    ─
20 |████████ ████████ ████████ ████       40%
   |████████ ████████ ████████ ████
10 |████████ ████████ ████████ ████ ███   20%
   |████████ ████████ ████████ ████ ███
 0 └────────┴────────┴────────┴────┴───→
     キズ    寸法不良  変形    汚れ  その他
     (45%)   (25%)   (15%)   (10%) (5%)
```

- **棒グラフ**: 各項目の件数（多い順に並べる）
- **折れ線グラフ**: 累積割合

### 3.3 パレート図の読み方

上の例では：
- **キズが全体の45%** を占めている
- キズ + 寸法不良で **70%** を占めている
- → **まずキズ対策を優先すべき**

### 3.4 パレートの法則（80:20の法則）

> 「結果の80%は、原因の20%から生じる」

品質管理では：
- 不良の80%は、原因の20%から発生している
- → **上位2〜3項目に集中対策すれば効果的**

## 4. M3/M4（受入検査DB・工程不良DB）での使い方

### 4.1 管理図の活用

| 場面 | 何を見る | 管理図で何がわかる |
|------|---------|------------------|
| 受入検査 | サプライヤから届く部品の品質 | 「最近A社の部品、バラつき増えてない？」 |
| 工程不良 | 自社製造工程の不良率 | 「組立工程の不良、増加傾向じゃない？」 |

**例：受入検査での管理図**

```
サプライヤA社の部品寸法（週次）

     UCL ─────────────────────────────────
                                    ●  ← 今週：要注意
                               ●
     CL  ────●───●───●───●───●────────────
            ●       ●
     LCL ─────────────────────────────────
         1月  2月  3月  4月  5月  6月
```

→ 「5月から上昇傾向、A社に確認した方がいい」

### 4.2 パレート図の活用

| 場面 | 何を見る | パレート図で何がわかる |
|------|---------|----------------------|
| 受入検査 | 不合格理由の内訳 | 「寸法不良が最多、測定器を確認すべき」 |
| 工程不良 | 不良種別の内訳 | 「キズが6割、治具の見直しが先決」 |

## 5. 必要かどうかの判断基準

### 5.1 SPC・管理図が必要な場合

- サプライヤの品質傾向を**継続的に**把握したい
- 自社工程の不良傾向を**早期に**検知したい
- 「なんとなく増えてる気がする」を**データで証明**したい
- 品質改善活動を**数値ベース**で進めたい

### 5.2 不要な場合

- 記録するだけで十分（分析は後で考える）
- Excelでグラフ作れば事足りる
- 人が見て判断すればいい
- データ量が少なく、統計的分析の意味がない

---

## 6. まとめ

| ツール | 目的 | 一言 |
|--------|------|------|
| SPC | 異常の早期検知 | 不良が出る前に気づく |
| 管理図 | 傾向の可視化 | データをグラフで見える化 |
| パレート図 | 優先順位付け | どこから手をつけるか決める |

**M3/M4では**:
- 「分析が肝」なら、これらのツールは**必須**
- 「記録だけ」なら、**後から追加**でもOK
- → **ヒアリングで要否を確認**する
