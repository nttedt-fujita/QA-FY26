# ERP（Enterprise Resource Planning）全体像

製造業における「あるべき姿」を理解するための参考資料。

---

## ERPとは

企業の基幹業務（販売・調達・製造・在庫・会計・品質管理）を**一つの統合データベース**で管理し、部門間の情報をリアルタイムに連携させるシステム。「一度入力すれば全社で使える」が基本思想。

---

## 主要モジュール

| モジュール | 略称 | 役割 |
|-----------|------|------|
| 販売管理 | SD | 受注・見積・出荷・請求 |
| 購買管理 | MM | 発注・入荷・仕入先評価 |
| 在庫管理 | IM | 在庫数量・ロケーション・棚卸 |
| 生産管理 | PP | 生産計画・製造指示・BOM |
| MRP | MRP | 所要量計算（いつ・何を・何個発注するか） |
| 品質管理 | QM | 受入検査・工程検査・不良管理・CAPA |
| 会計 | FI/CO | 財務会計・管理会計・原価計算 |

---

## 製造業の典型的な業務フロー

```
受注 → 生産計画 → MRP → 調達 → 受入検査(IQC) → 製造 → 工程検査(PQC) → 完成品検査(OQC) → 出荷 → 売上
```

| 工程 | ERPの役割 |
|------|----------|
| 受注 | 受注伝票→生産計画へ連携。在庫・生産能力参照で納期回答 |
| 生産計画 | PSI計画に基づく生産指示。BOM展開で必要部材算出 |
| MRP | 在庫数 + 発注残 − 所要量 = 不足分を算出し発注推奨を生成 |
| 調達 | 発注書自動生成、納期管理、仕入先実績蓄積 |
| 受入検査（IQC） | 検査ロット自動生成、合否判定→合格品のみ在庫計上 |
| 製造 | 製造指示に対する実績入力、部材引当、工数記録 |
| 工程検査（PQC） | 検査データ記録、不合格時のフロー管理（手直し/廃棄） |
| 完成品検査（OQC） | 合格→出荷可能在庫へステータス変更 |
| 出荷 | 出荷伝票、ロット番号による出荷先追跡 |
| 売上 | 出荷と連動して売上伝票・請求書を自動生成 |

## 品質管理（QM）モジュールの詳細

### 受入検査（IQC）

| 機能 | 説明 |
|------|------|
| 検査ロット自動生成 | 入荷時に購買オーダーと紐づいた検査ロットが自動作成 |
| 検査計画 | 部品カテゴリごとに検査項目・基準を事前定義（マスタ管理） |
| 抜取検査 | AQL基準に基づく抜取数の自動決定 |
| 合否判定 | 合格→在庫計上、不合格→保留在庫→返品/特採判断 |
| 仕入先評価 | 不合格率を仕入先別・部品別に自動集計 |

### 工程内検査（PQC）

| 機能 | 説明 |
|------|------|
| 工程別検査ポイント | 製造工程のどのタイミングで検査するかを定義 |
| SPC（統計的工程管理） | 管理図で工程バラつき監視。異常検知時にアラート |
| 不合格品フロー | 手直し→再検査 / 廃棄 / 等級下げのルーティング |

### 不良管理

| 機能 | 説明 |
|------|------|
| 不良コード体系 | 不良種別をコード化（外観/寸法/機能不良等） |
| 原因分析 | 原因コード・対策を記録。8Dレポートとの連携 |
| CAPA管理 | 是正処置/予防処置の進捗追跡 |
| コスト計上 | 廃棄・手直しコストを原価に反映 |

### ロット管理・トレーサビリティ

| 機能 | 説明 |
|------|------|
| ロット番号管理 | 入荷ロット→製造ロット→出荷ロットの紐づけ |
| 正方向トレース | 「この部材ロットはどの完成品に使われたか？」 |
| 逆方向トレース | 「この完成品にはどのロットの部材が使われたか？」 |

## SCM（サプライチェーン管理）との連携

### PSI計画

```
        4月    5月    6月    7月
販売計画  40     45     50     45
生産計画  45     50     50     40
在庫推移  15     20     20     15
```

ERPの役割: PSI計画を一元管理し、変更があれば即座にMRP→発注計画に反映。

### 在庫管理のキー概念

| 概念 | 説明 |
|------|------|
| 在庫ステータス | 利用可能 / 保留（検査中）/ 不良 / 予約済み |
| 引当 | 製造指示に対して在庫を「予約」 |
| 安全在庫 | リードタイムと需要変動から最低在庫量を設定 |

## 御社の現状との対比

| ERPモジュール | 御社の現状 | ギャップ |
|-------------|----------|---------|
| 部材管理（MM/IM） | GEN | ✅ あり（品質情報なし） |
| 完成品管理 | kintone | ✅ あり（ロット紐づけなし） |
| PSI計画 | Excel（掛川さん手動） | ⚠️ リアルタイム性なし |
| 品質管理（QM） | なし | ❌ M3/M4で補完予定 |
| MRP | 鶴田さん独自システム | ⚠️ 属人的 |
| 統合連携 | 手動転記 | ❌ 島と島をヒトの手で接続 |

### 御社の「虫食い状態」の構造

```
現状:
  [GEN] ←手動転記→ [Excel PSI] ←手動転記→ [kintone]
                                              品質データなし

あるべき姿（ERP的統合）:
  部材入荷（ロット付き）→ 受入検査 → 在庫計上 → 製造引当 →
  工程検査 → 完成品登録 → 出荷 → 全てトレース可能
```

## 年400-500台規模でのERP導入判断

**事実**: フルスペックERP（SAP等）は導入コスト数千万〜。年400-500台規模ではROIが見合わない可能性が高い。

**現実的な選択肢**（要検討）:
1. 現在のGEN + kintone + Excelの「つなぎ」を改善
2. 品質管理（QM）部分をM3/M4で補完
3. 将来的にGEN・kintoneと連携できる設計にする
